Get Instance Field Value Instruction

{link:getfield} - 0xb4 (180)

Purpose

Fetches the value of an instance field from an object reference and pushes it onto the operand stack.

The field is selected by resolving a symbolic field reference from the run-time constant pool.

Operands:
  • FieldDescriptor: Specifies the class, field name, and type descriptor of the instance field to fetch. The format is Owner->FieldName:TypeDescriptor. The Owner does not include the L and ; typically used in type descriptors.
Format:

getfield <FieldDescriptor>

Examples:
  • getfield java/lang/String->value:[C
  • getfield java/util/ArrayList->size:I
TypeDescriptor:

A TypeDescriptor specifies the type of the field. It must represent a valid Java type:

  • I: int
  • J: long
  • F: float
  • D: double
  • Z: boolean
  • B: byte
  • C: char
  • S: short
  • L<classname>;: A reference type (e.g., Ljava/lang/String;)
  • [: An array type (e.g., [I for int[])
Stack Effect:
  • Before: ..., objectref
  • After: ..., value (the field value; category 1 for most types, category 2 for long/double)
Verification:
  • Verification requires a reference-compatible input and guarantees a value of the field type for successor frames.
  • If type resolution fails, the corresponding linkage error is thrown.
Runtime Exceptions and Errors:
  • NullpointerException: if the object reference is null.
  • IncompatibleClassChangeError: if the resolved field is static.
  • Linkage errors (such as NoSuchFieldError, IllegalAccessError, or NoClassDefFoundError): if the symbolic field reference cannot be resolved or if the resolved field is not accessible.
