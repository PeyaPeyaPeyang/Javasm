Array Store Instructions

{link:iastore} - 0x4f (79)
{link:lastore} - 0x50 (80)
{link:fastore} - 0x51 (81)
{link:dastore} - 0x52 (82)
{link:aastore} - 0x53 (83)
{link:bastore} - 0x54 (84)
{link:castore} - 0x55 (85)
{link:sastore} - 0x56 (86)

Purpose

Stores a value into an array. The array reference, index, and the value to store are popped from the operand stack.

  • {link:iastore}: stores int into int[]
  • {link:lastore}: stores long into long[]
  • {link:fastore}: stores float into float[]
  • {link:dastore}: stores double into double[]
  • {link:aastore}: stores reference into Object[]
  • {link:bastore}: stores byte or boolean into byte[]/boolean[]
  • {link:castore}: stores char into char[]
  • {link:sastore}: stores short into short[]

If the array reference is null or the index is out of bounds, an exception is thrown.

For {link:aastore}, the stored reference must be assignment-compatible with the runtime component type of the destination array.

Operands: (none)
Format: {instruction}
Examples:
  • iastore
  • lastore
  • fastore
  • dastore
  • aastore
  • bastore
  • castore
  • sastore
Stack Effect:
  • iastore, fastore, aastore, bastore, castore, sastore
    Before: ..., arrayref, index, value
    After: ...
  • lastore, dastore
    Before: ..., arrayref, index, value (value is category-2)
    After: ...
    (Note: long and double values occupy two stack slots)
Verification:
  • Verification requires that the operand stack can provide the appropriate types for the array reference, index, and value at the point of this instruction.
  • For {link:aastore}, verification checks that the value is a reference type and that the array reference is compatible with an array of that reference type.
Notes:
  • These instructions do not resolve constant-pool entries and have no instruction-specific linkage errors.
  • For category-2 value types (long, double), operand-stack slot accounting is two words for value plus one for arrayref and one for index.
Runtime Exceptions and Errors:
  • NullPointerException if arrayref is null.
  • ArrayIndexOutOfBoundsException if index is out of range.
  • ArrayStoreException for aastore when the value is not assignment-compatible.
