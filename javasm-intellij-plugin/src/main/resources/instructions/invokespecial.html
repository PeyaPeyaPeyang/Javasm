Invoke Special Method

{link:invokespecial} - 0xb7 (183)

Purpose

Invokes methods using special (non-virtual) invocation rules:
  • Instance constructors (<init>)
  • Private instance methods
  • Explicit superclass method invocations (for example super.m())
This instruction bypasses normal virtual dispatch lookup.
Operands:
  • methodref (2 bytes): Index into the constant pool, pointing to a Methodref
Format:
invokespecial <methodref>
Methodref format:

The method reference format specifies the target method as follows:

<classname>-><methodname>(<argument_descriptor>)<return_descriptor>
  • <classname>: Fully qualified class name in JVM internal form (slash-separated), e.g., java/lang/Object
  • <methodname>: instance method name or <init> (constructor)
  • <argument_descriptor>: JVM method argument descriptor
  • <return_descriptor>: JVM return type descriptor
Examples:
  • invokespecial java/lang/Object-><init>()V
  • invokespecial my/pkg/Base->doWork()V (super invocation)
  • invokespecial my/pkg/MyClass->privateHelper(I)V
Stack Effect:

Before: ..., objectref, arg1, arg2, ..., argN
After: ..., returnValue?

Pops receiver plus all arguments. Pushes return value if method return type is non-void. Constructor calls use return type V.

Verification:
  • Verification enforces constructor rules for uninitialized objects and legal <init> usage.
Notes:
  • <clinit> is not invoked by bytecode call instructions; class initialization is triggered implicitly.
  • Use invokespecial when semantic intent requires non-virtual selection.
Runtime Exceptions and Errors:
  • NullPointerException: if the operand stack is empty (i.e., no object reference to invoke on).
  • LinkageError: if the symbolic method reference cannot be resolved or if the resolved method is not accessible.
  • IncompatibleClassChangeError: if the resolved method is static, if an instance method is resolved for a constructor reference, or if the resolved method is not a valid method handle for the invocation type.
  • IllegalAccessError: if the resolved method is an instance method that is not accessible from the current class.
