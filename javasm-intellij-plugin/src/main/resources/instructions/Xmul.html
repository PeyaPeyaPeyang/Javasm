Multiplication Instructions

{link:imul} - 0x68 (104)
{link:lmul} - 0x69 (105)
{link:fmul} - 0x6a (106)
{link:dmul} - 0x6b (107)

Purpose

Multiplies the top two operands on the operand stack and pushes the result. The operation respects the type of the instruction.

  • {link:imul}/{link:lmul}: two's-complement multiplication with wraparound on overflow.
  • {link:fmul}/{link:dmul}: IEEE 754 multiplication (NaN propagation, signed zeros, infinities).
Operands: None
Format: {instruction}
Examples:
  • imul
  • dmul
Stack Effect:
  • {link:imul}, {link:fmul}
    Before: ..., value1, value2 (category-1)
    After: ..., result (category-1)
  • {link:lmul}, {link:dmul}
    Before: ..., value1, value2 (category-2 values)
    After: ..., result (category-2)
Notes:
  • {link:imul} and {link:lmul} perform two's-complement multiplication. If the mathematical result exceeds the range of the type, it wraps around modulo 2^32 for imul and modulo 2^64 for lmul.
  • {link:fmul} and {link:dmul} follow IEEE 754 rules for floating-point multiplication, including handling of NaN, infinities, and signed zeros.
  • These instructions do not resolve constant-pool entries and have no instruction-specific runtime exceptions.
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values at the point of this instruction (two category-1 values for imul and fmul, two category-2 values for lmul and dmul).
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
