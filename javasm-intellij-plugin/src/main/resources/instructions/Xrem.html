Remainder Instructions

{link:irem} - 0x70 (112)
{link:lrem} - 0x71 (113)
{link:frem} - 0x72 (114)
{link:drem} - 0x73 (115)

Purpose

Computes the remainder of dividing the second-top operand by the top operand and pushes the result. Integer division by zero throws ArithmeticException. Floating-point operations follow IEEE 754 semantics.

  • {link:irem}/{link:lrem}: result sign follows dividend; division-by-zero is exceptional.
  • {link:frem}/{link:drem}: IEEE 754 remainder behaviour, including NaN propagation.
Operands: None
Format: {instruction}
Examples:
  • irem
  • drem
Stack Effect:
  • {link:irem}, {link:frem}
    Before: ..., value1, value2 (category-1; value1 is dividend)
    After: ..., remainder (category-1)
  • {link:lrem}, {link:drem}
    Before: ..., value1, value2 (category-2 values; value1 is dividend)
    After: ..., remainder (category-2)
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values at the point of this instruction.
Runtime Exceptions:
  • ArithmeticException: if the divisor (top operand) is zero for irem or lrem.
