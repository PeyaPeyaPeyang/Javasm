Store Instance Field Instruction

{link:putfield} - 0xb5 (181)

Purpose

Stores a value into an instance field of an object. The object reference and the value are popped from the operand stack, and the value is assigned to the specified instance field.

The symbolic field reference is resolved from the run-time constant pool before assignment.

Operands:
  • FieldDescriptor: Specifies the class, field name, and type descriptor of the instance field to store the value into. The format is Owner->FieldName:TypeDescriptor. The Owner does not include the L and ; typically used in type descriptors.
Format:

putfield <FieldDescriptor>

Examples:
  • putfield java/lang/String->value:[C
  • putfield java/util/ArrayList->size:I
TypeDescriptor:

A TypeDescriptor specifies the type of the field. It must represent a valid Java type:

  • I: int
  • J: long
  • F: float
  • D: double
  • Z: boolean
  • B: byte
  • C: char
  • S: short
  • L<classname>;: A reference type (e.g., Ljava/lang/String;)
  • [: An array type (e.g., [I for int[])
Stack Effect:
  • Before: ..., objectref, value (value may be category-1 or category-2 by descriptor)
  • After: ... (consumes value and object reference)
Verification:
  • Verification requires a reference-compatible object reference and a value compatible with the field type on the operand stack at the point of this instruction.
  • If type resolution fails, the corresponding linkage error is thrown.
Notes:
  • For category-2 field types (long, double), operand-stack slot accounting is two words for value plus one for objectref.
  • Verifier checks value-type assignment compatibility with the field descriptor at each control-flow point.
Runtime Exceptions and Errors:
  • NullPointerException may also be thrown if the operand stack does not contain enough values for the field type (e.g., missing value or object reference).
  • IncompatibleClassChangeError may also be thrown if the resolved field is not an instance field (e.g., static field).
  • NoSuchFieldError may also be thrown if the field name or type does not match any field in the resolved class.
  • IllegalAccessError may also be thrown if the field is not accessible from the current class (e.g., private field in another class).
  • NoClassDefFoundError may also be thrown if the class containing the field cannot be found at runtime.
