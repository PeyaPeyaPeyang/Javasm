Range-Based Switch Instruction

{link:tableswitch} - 0xaa (170)

Purpose

Performs a switch-style jump based on the int key on the operand stack. The key is matched against a continuous range starting from the specified low index. Execution jumps to the corresponding label based on the key's offset from the low index. If the key is out of range, control jumps to the default label.

Operands:
  • Padding bytes (0x00) so that the first 32-bit field starts at a 4-byte boundary after opcode.
  • default: signed 32-bit branch offset relative to the opcode address.
  • low, high: signed 32-bit key bounds (inclusive).
  • jump offsets: high - low + 1 signed 32-bit offsets, one for each key in order.
Format:
tableswitch <low_index> {
  label_0;
  label_1;
  ...
} default <default_label>
Examples:
tableswitch 1 {
  case_one;
  case_two;
  case_three;
} default default_case;
Stack Effect:
  • Before: ..., key (an int value)
  • After: ... (consumes one category-1 int, then branches)

If the key is outside the continuous range starting from low_index to low_index + number_of_labels - 1, the control jumps to the default label, which is mandatory.

Verification:
  • All computed targets (default and case targets) must point to valid instruction boundaries in the same method.
  • Frame types at all switch targets must be merge-compatible.
Notes:
  • tableswitch is optimal for dense key ranges; sparse keys are usually better served by lookupswitch.
