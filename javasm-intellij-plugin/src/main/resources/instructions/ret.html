Return from Subroutine

{link:ret} - 0xa9 (169)

Purpose

The ret instruction is used to return from a subroutine. It retrieves the return address from a local variable and transfers control to that address.

This instruction is part of the legacy jsr/ret subroutine model historically used for old finally compilation patterns.

Operands:
  • index: local variable index containing a value of the special returnAddress type. In bytecode encoding, this is an unsigned 8-bit index.
Format: ret <index>
Notes on wide prefix:

With the wide prefix, the operand index is extended from 1 byte to 2 bytes, allowing local variable indices larger than 255.

Examples:
  • ret 1
  • ret 0x05
  • wide ret 0x0100 (index 256)
Stack Effect:
  • Before: ...
  • After: ... (no operand stack change)
Verification:
  • Verification requires local variable index to contain a valid returnAddress value.
  • Control-flow and stack/local type states must remain consistent across all branch targets.
Notes:
  • ret is obsolete in modern class-file workflows and is generally considered legacy-only.
  • Use structured exception handlers for finally-style logic in contemporary bytecode instead of legacy jsr/ret patterns.
  • Modern verification strategies centered on stack-map frames MAY REJECT legacy subroutine patterns.
