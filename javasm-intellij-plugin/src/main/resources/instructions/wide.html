Widening Instruction

{link:wide} - 0xc4 (196)

Purpose

Modifies specific local-variable instructions so their local-variable index is encoded as 16 bits. For iinc, it also widens the increment constant to signed 16 bits.

Operands:
  • Instruction: one of iload, fload, aload, lload, dload, istore, fstore, astore, lstore, dstore, ret, or iinc.
  • Index: unsigned 16-bit local-variable index.
  • Const: signed 16-bit increment value (only for wide iinc).
Format:

wide <Instruction> <Index> [const]

Examples:
  • wide iload 300 (loads a local variable at index 300)
  • wide iinc 300 5 (increments the local variable at index 300 by 5)
Supported Instructions:

The following instructions can be modified by wide:

  • iload
  • fload
  • aload
  • lload
  • dload
  • istore
  • fstore
  • astore
  • lstore
  • dstore
  • ret
  • iinc
Stack Effect:

The stack effect depends on the instruction being modified by wide. For example:

  • wide iload:
    Before: ...
    After: ..., value
  • wide istore:
    Before: ..., value
    After: ...
Verification:
  • wide does not change instruction semantics, only operand encoding width.
  • Using wide with unsupported opcodes is invalid class-file structure.
Runtime Exceptions and Errors:

wide itself does not throw any instruction-specific runtime exceptions. However, the modified instruction may throw exceptions as per its normal semantics (for example, wide iload may throw NullPointerException if loading a reference from an uninitialized local variable).
