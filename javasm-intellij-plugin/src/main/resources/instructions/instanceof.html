Check Instance Type Instruction

{link:instanceof} - 0xc1 (193)

Purpose

Tests whether the object reference on top of the operand stack is an instance of the specified reference type. Pushes 1 if true, 0 if false.

The target type is obtained by resolving a symbolic class reference from the run-time constant pool.

Operands:
  • TypeDescriptor: constant-pool class reference naming a class, interface, or array type (e.g., Ljava/lang/String;, [Ljava/lang/Object;). Primitive targets are not valid.
Format: instanceof <TypeDescriptor>
Examples:
  • instanceof Ljava/lang/String;
  • instanceof Ljava/util/List;
Stack Effect:
  • Before: ..., objectref
  • After: ..., result (category-1 int: 0 or 1)
Semantics:
  • If objectref is null, result is 0 (no exception is thrown).
  • If non-null, result is 1 iff the runtime class is assignment-compatible with target type.
  • For array targets, normal JVM array assignability rules apply.
Notes:
  • Typical pattern: instanceof T followed by conditional branch and checkcast T.
  • Use this instruction for safe type tests before value-dependent casts in polymorphic code.
Verification:
  • Verification requires a reference-compatible input and guarantees an int result for successor frames.
  • If type resolution fails, the corresponding linkage error is thrown.
  • There is no ClassCastException path in instanceof; failed checks return 0.
Runtime Exceptions and Errors:
  • NullPointerException: if the operand stack is empty (i.e., no object reference to check).
  • LinkageError: if the symbolic type reference cannot be resolved or if the resolved type is not accessible.
