Create New Object Instruction

{link:new} - 0xbb (187)

Purpose

Allocates memory for a new object of the specified class. The object is not initialized; the constructor must be invoked explicitly using the invokespecial instruction.

The pushed value is an uninitialized object reference tracked by the verifier until a matching constructor call completes.

Operands:
  • ClassRef: constant-pool index to CONSTANT_Class_info naming a non-array, non-interface class.
Format:

new <ClassRef>

Examples:
  • new java/lang/String followed by invokespecial java/lang/String-><init>()V
  • new java/util/ArrayList followed by its constructor invocation
Class Reference Constraints:
  • Must resolve to a class type, not an array type and not an interface.
  • Class access checks and loading/linking rules apply at resolution time.
Stack Effect:

The operand stack is updated with an uninitialized reference to the newly allocated object.

  • Before: ...
  • After: ..., uninitialized-objectref (category-1 reference)
Verification:
  • Verification requires that the operand stack can accept a reference value at the point of this instruction.
  • Verification tracks uninitialized references to ensure proper constructor invocation patterns.
  • If type resolution fails, the corresponding linkage error is thrown.
Notes:
  • The verifier enforces correct constructor initialisation flow before normal use of the reference.
  • Object creation is logically separate from constructor execution.
Runtime Exceptions and Errors:
  • OutOfMemoryError: if the Java Virtual Machine cannot allocate the object because it is out of memory.
  • LinkageError: if the symbolic class reference cannot be resolved or if the resolved class is not accessible.
  • InstantiationError: if the resolved class is abstract or an interface.
  • IncompatibleClassChangeError: if the resolved class is not a class (e.g., if it's an array or interface) or if it violates other constraints.
