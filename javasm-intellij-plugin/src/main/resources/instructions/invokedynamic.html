Invoke Dynamic

{link:invokedynamic} - 0xba (186)

Purpose

Performs a dynamic call-site invocation whose target is linked by a bootstrap method at runtime. It is the foundation for Java lambda translation and language-level dynamic dispatch mechanisms.

Unlike the other invoke instructions, the call target is not fixed solely by a class/method symbolic reference. Instead, linkage produces a CallSite whose target method handle is then invoked.

Operands:
  • Bytecode operand is a constant-pool index to CONSTANT_InvokeDynamic_info, followed by two zero bytes.
  • CONSTANT_InvokeDynamic_info links to a bootstrap method entry and a name-and-type descriptor.
  • Bootstrap static arguments are encoded in the class-file BootstrapMethods attribute.
Format:
invokedynamic <name> <methodDescriptor> [bootstrap = MethodHandle + static args]
Details:
  • At first execution, the JVM resolves the call site and invokes the bootstrap method.
  • The bootstrap method receives lookup context, call-site name, method type, and static arguments.
  • It must return a non-null CallSite whose target type exactly matches the call-site descriptor.
  • After successful linkage, subsequent executions reuse the linked call site.
Example:
invokedynamic apply (Ljava/lang/Object;)Ljava/lang/Object;
  // bootstrap: java/lang/invoke/LambdaMetafactory.metafactory
Stack Effect:

Before: ..., arg1, arg2, ..., argN
After: ..., returnValue
Arguments and return type are defined by methodDescriptor. Category-2 values (long/double) consume or produce two slots.

Verification:
  • Verification checks operand-stack compatibility with the call-site descriptor.
  • Linkage failures are reported as linkage errors (commonly BootstrapMethodError wrappers).
  • If the linked target cannot be invoked with actual argument types, method-handle invocation type errors may occur.
  • No objectref is implicit in this instruction; all arguments are explicit in the descriptor.
Notes:
  • Use invokedynamic when call behaviour is intentionally deferred to bootstrap logic.
  • In Java compilation, lambda expressions and string concatenation are common producers of this opcode.
Runtime Exceptions and Errors:
  • BootstrapMethodError: if the bootstrap method fails to link the call site (e.g., throws an exception, returns null, or returns a CallSite with an incompatible target type).
  • LinkageError: if the symbolic reference cannot be resolved or if the resolved method is not accessible.
  • IncompatibleClassChangeError: if the resolved method handle is not a valid method handle or if the resolved call site is not a valid call site.
  • NullPointerException: if the operand stack does not contain enough arguments for the call site descriptor.
  • WrongMethodTypeException: if the actual argument types do not match the call site descriptor at runtime.
