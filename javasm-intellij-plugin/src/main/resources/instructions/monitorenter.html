Acquire Monitor Lock

{link:monitorenter} - 0xc2 (194)

Purpose

Acquires the monitor lock for the object referenced on top of the operand stack. Used to enter a synchronized block or method.

If the current thread already owns the monitor, entry is reentrant and the monitor's hold count is incremented.

Operands: None
Format: monitorenter
Examples:
  • monitorenter
Stack Effect:
  • Before: ..., objectref
  • After: ... (consumes one category-1 reference)
Runtime and Concurrency Notes:
  • Monitor acquisition may block until ownership becomes available.
  • Successful monitorenter establishes synchronisation ordering as defined by the Java Memory Model.
  • No constant-pool resolution is involved in this instruction.
Usage Note

monitorenter should always be paired with monitorexit to properly release the acquired monitor lock and avoid deadlocks. Typically, monitorenter is used at the start of a synchronized block, and monitorexit at the end.

In compiler-generated patterns, a try/finally-equivalent shape ensures the lock is released on both normal and exceptional exits.

Verification:
  • Verification requires that the operand stack can accept a reference value at the point of this instruction.
Runtime Exceptions and Errors:
  • NullPointerException: if the object reference on top of the stack is null.
