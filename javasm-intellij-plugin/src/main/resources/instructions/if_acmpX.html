Reference Comparison Instructions

{link:if_acmpeq} - 0xa5 (165)
{link:if_acmpne} - 0xa6 (166)

Purpose

Compares two reference values for reference identity (same object or both null) and conditionally branches.

  • {link:if_acmpeq}: branches when the two references are identical.
  • {link:if_acmpne}: branches when the two references are not identical.

This is not content equality. It does not call equals().

Operands:
  • label: target label encoded as a signed 16-bit offset relative to the current opcode.
Format:
  • if_acmpeq <label>
  • if_acmpne <label>
Examples:
  • if_acmpeq same_ref (true for same object reference, including both null)
  • if_acmpne different_ref (true for different object identities)
Stack Effect:
  • Before: ..., value1 (reference), value2 (reference)
  • After: ... (consumes two category-1 reference values)
Verification:
  • Verification requires that the operand stack can accept two category-1 reference values at the point of this instruction.
  • Branch target and frame-type consistency are verified at class loading/linking time.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions. Invalid branch targets are rejected during verification.
