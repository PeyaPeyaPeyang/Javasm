Release Monitor Lock Instruction

{link:monitorexit} - 0xc3 (195)

Purpose

Releases the monitor lock for the object referenced on top of the operand stack. Used to exit a synchronized block or method.

The current thread must own the monitor. For reentrant locking, each monitorexit decrements hold count and releases the monitor only when the count reaches zero.

Operands: None
Format: monitorexit
Examples:
  • monitorexit
Stack Effect:
  • Before: ..., objectref
  • After: ... (consumes one category-1 reference)
Runtime and Concurrency Notes:
  • If the current thread does not own the monitor, IllegalMonitorStateException is thrown.
  • Successful monitor release participates in Java Memory Model synchronization ordering.
  • No constant-pool resolution is involved in this instruction.
Usage Note

monitorexit must always be paired with monitorenter to properly release the monitor lock acquired earlier. Failure to do so may cause deadlocks or inconsistent synchronization.

Correct bytecode ensures monitor release on all paths, including exceptional control flow.

Verification:
  • Verification requires that the operand stack can accept a reference value at the point of this instruction.
Runtime Exceptions:
  • IllegalMonitorStateException: if the current thread does not own the monitor for the object reference on top of the stack.
