Table-Based Switch Instruction

{link:lookupswitch} - 0xab (171)

Purpose

Performs a switch-style jump based on the top int value on the operand stack. It matches the key against a set of match-offset pairs and jumps to the corresponding label. If no match is found, jumps to the default label.

Operands:
  • A list of int key and label pairs, plus a mandatory default label.
Format:
lookupswitch {
  key_1: label_1;
  key_2: label_2;
  ...
  default: label_default;
}

Keys can be specified in decimal or hexadecimal (e.g., 0x00), and labels are symbolic jump targets.

Examples:
lookupswitch {
  0: case_zero;
  1: case_one;
  default: default_case;
}
Stack Effect:
  • Before: ..., key (an int value)
  • After: (no values; control flow jumps to the matched label)

If the key does not match any listed case, the control jumps to the default label, which is mandatory.
