Table-Based Switch Instruction

{link:lookupswitch} - 0xab (171)

Purpose

Performs a switch-style jump based on the top int value on the operand stack. It matches the key against a set of match-offset pairs and jumps to the corresponding label. If no match is found, jumps to the default label.

Operands:
  • Padding bytes (0x00) so that the first 32-bit field starts at a 4-byte boundary after the opcode.
  • default: signed 32-bit branch offset relative to the opcode address.
  • npairs: signed 32-bit count of match-offset pairs (must be non-negative).
  • match_i/offset_i: signed 32-bit key and branch offset pairs.
  • The match_i keys must appear in strictly increasing numeric order.
Format:
lookupswitch {
  key_1: label_1;
  key_2: label_2;
  ...
  default: label_default;
}

Keys can be specified in decimal or hexadecimal (e.g., 0x00), and labels are symbolic jump targets.

Examples:
lookupswitch {
  0: case_zero;
  1: case_one;
  default: default_case;
}
Stack Effect:
  • Before: ..., key (an int value)
  • After: ... (consumes one category-1 int, then branches)

If the key does not match any listed case, the control jumps to the default label, which is mandatory.

Verification:
  • All branch targets must point to valid instruction boundaries in the same method.
  • Control-flow merges from all switch targets must satisfy local/stack type consistency.
  • No constant-pool resolution is performed by this opcode.
Notes:
  • Use lookupswitch for sparse key sets; dense key intervals are usually encoded as tableswitch.
  • Bytecode tooling must preserve required padding and key sort order.
Runtime Exceptions and Errors:

This instruction does not throw any instruction-specific runtime exceptions. Invalid branch targets are rejected during verification.
