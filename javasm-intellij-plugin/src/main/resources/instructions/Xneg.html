Negate Instructions

{link:ineg} - 0x74 (116)
{link:lneg} - 0x75 (117)
{link:fneg} - 0x76 (118)
{link:dneg} - 0x77 (119)

Purpose

Negates the top value on the operand stack (i.e., multiplies by -1) and pushes the result back on the stack.

  • {link:ineg}/{link:lneg}: two's-complement negation; MIN_VALUE negates to itself.
  • {link:fneg}/{link:dneg}: IEEE 754 sign inversion (including +0.0 to -0.0).
Operands: None
Format: {instruction}
Examples:
  • ineg
  • dneg
Stack Effect:
  • {link:ineg}, {link:fneg}
    Before: ..., value (category-1)
    After: ..., -value (category-1)
  • {link:lneg}, {link:dneg}
    Before: ..., value (category-2)
    After: ..., -value (category-2)
Notes:
  • {link:ineg} and {link:lneg} perform two's-complement negation, so negating the minimum value (e.g. Integer.MIN_VALUE) results in the same value due to overflow.
  • {link:fneg} and {link:dneg} perform IEEE 754 sign inversion, so negating +0.0 results in -0.0, and negating -0.0 results in +0.0.
  • These instructions do not resolve constant-pool entries and have no instruction-specific runtime exceptions.
Verification:
  • Verification requires that the operand stack can accept the appropriate category of value at the point of this instruction.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
