Purpose

Pushes a specific constant value of a given primitive type onto the operand stack.

  • iconst_m1 to iconst_5: push int constants -1 through 5.
  • lconst_0, lconst_1: push long 0 and 1.
  • fconst_0, fconst_1, fconst_2: push float 0.0, 1.0, 2.0.
  • dconst_0, dconst_1: push double 0.0 and 1.0.
Operands: None
Format: {instruction}
Examples:
  • iconst_3 (pushes int 3)
  • lconst_1 (pushes long 1)
  • dconst_1 (pushes double 1.0)
  • fconst_0 (pushes float 0.0)
Stack Effect:
  • Before: ...
  • After: ..., value (category-1 for int/float, category-2 for long/double)
Notes:
  • These instructions are optimised for frequently used constants, avoiding the need for constant pool references for these values.
  • Use bipush for byte-range constants and sipush for short-range constants. For other values, use ldc or related instructions to load from the constant pool.
  • These instructions do not resolve constant-pool entries and have no instruction-specific runtime exception.
Verification:
  • Verification requires that the operand stack can accept the appropriate category of value for each instruction at the point of this instruction.
