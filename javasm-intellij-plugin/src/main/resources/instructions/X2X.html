Purpose

Converts the value on top of the operand stack from one primitive type to another. The value is popped, converted, and the result is pushed back onto the stack.

Conversion Semantics:
  • Floating to integer (f2i, f2l, d2i, d2l) rounds toward zero.
  • For floating-to-integer conversion, NaN maps to zero.
  • For out-of-range floating values, the result is clamped to the target type min/max.
  • i2b, i2s, and i2c perform narrowing conversion from int; i2c zero-extends to unsigned 16-bit char.
  • f2d is widening; d2f may lose precision and may produce infinities.
Operands: None
Format: {instruction}
Examples:
  • d2i
  • i2l
  • f2d
Stack Effect:
  • Before: ..., value (type1)
  • After: ..., value (type2)
Category/Slot Notes:
  • Conversions involving long/double consume or produce category-2 values (two stack slots).
  • All other primitive conversion results are category-1.
Verification:
  • Verification requires that the operand stack can accept the input type and output type at the point of this instruction.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
