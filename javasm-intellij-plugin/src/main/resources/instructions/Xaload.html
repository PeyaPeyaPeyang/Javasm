Array Load Instructions

{link:iaload} - 0x2e (46)
{link:laload} - 0x2f (47)
{link:faload} - 0x30 (48)
{link:daload} - 0x31 (49)
{link:aaload} - 0x32 (50)
{link:baload} - 0x33 (51)
{link:caload} - 0x34 (52)
{link:saload} - 0x35 (53)

Purpose

Loads a value from an array onto the operand stack. The array reference and index are popped from the stack, and the value at the given index is pushed.

  • {link:iaload}: loads int from int[]
  • {link:laload}: loads long from long[]
  • {link:faload}: loads float from float[]
  • {link:daload}: loads double from double[]
  • {link:aaload}: loads reference from Object[]
  • {link:baload}: loads byte or boolean from byte[]/boolean[]
  • {link:caload}: loads char from char[]
  • {link:saload}: loads short from short[]

baload sign-extends byte values to int, while boolean array loads produce 0 or 1. caload zero-extends char to int, and saload sign-extends short to int.

If the array reference is null or the index is out of bounds, an exception is thrown.

Operands: (none)
Format: {instruction}
Examples:
  • iaload
  • laload
  • faload
  • daload
  • aaload
  • baload
  • caload
  • saload
Stack Effect:
  • iaload, faload, aaload, baload, caload, saload
    Before: ..., arrayref, index
    After: ..., value
  • laload, daload
    Before: ..., arrayref, index
    After: ..., value (category-2)
    (Note: long and double occupy two stack slots)
Verification:
  • Verification requires that the operand stack can accept a reference and an int value for the array reference and index, respectively, at the point of this instruction.
Runtime Exceptions:
  • NullPointerException if arrayref is null.
  • ArrayIndexOutOfBoundsException if index is out of range.
