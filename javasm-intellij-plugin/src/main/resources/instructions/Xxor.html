Bitwise XOR Instructions

{link:ixor} - 0x82 (130)
{link:lxor} - 0x83 (131)

Purpose

Performs a bitwise exclusive OR (XOR) operation on the top two values of the operand stack. The result is pushed back onto the operand stack.

  • {link:ixor}: operates on two int values
  • {link:lxor}: operates on two long values
Operands: None
Format: {instruction}
Examples:
  • ixor
  • lxor
Stack Effect:
  • ixor
    Before: ..., value1 (int), value2 (int) (category-1)
    After: ..., result (int) (category-1)
    Operation: value1 ^ value2
  • lxor
    Before: ..., value1 (long), value2 (long) (category-2)
    After: ..., result (long) (category-2)
    Operation: value1 ^ value2
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values at the point of this instruction.
Notes:
  • The XOR operation is performed independently on each corresponding bit of the operands. The result bit is 1 if the operand bits differ, and 0 if they are the same.
  • The operation treats values purely as bit patterns; it is independent of signed or unsigned interpretation.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
