Division Instructions

{link:idiv} - 0x6c (108)
{link:ldiv} - 0x6d (109)
{link:fdiv} - 0x6e (110)
{link:ddiv} - 0x6f (111)

Purpose

Divides the second-top operand on the stack by the top operand and pushes the result. For integer types, division by zero throws ArithmeticException. For floating types, division by zero results in Infinity or NaN per IEEE 754.

  • {link:idiv}/{link:ldiv}: integer division truncates toward zero.
  • {link:fdiv}/{link:ddiv}: IEEE 754 arithmetic (including NaN propagation and signed infinities/zeros).
  • The integer overflow corner case MIN_VALUE / -1 yields MIN_VALUE (wraparound semantics).
Operands: None
Format: {instruction}
Examples:
  • idiv
  • ddiv
Stack Effect:
  • Before: ..., value1, value2 (value1 is dividend, value2 is divisor)
  • After: ..., result (category follows instruction type)
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values for the instruction type at the point of this instruction.
Notes:
  • fdiv and ddiv follow IEEE 754 rules, so division by zero is not an exception but results in specific values (Infinity or NaN).
Runtime Exceptions:
  • ArithmeticException: thrown by idiv or ldiv if the divisor (top operand) is zero.
