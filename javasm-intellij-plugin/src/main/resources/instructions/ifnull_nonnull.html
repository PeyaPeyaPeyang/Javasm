Null Check Branch Instructions

{link:ifnull} - 0xc6 (198)
{link:ifnonnull} - 0xc7 (199)

Purpose

Tests whether the top operand-stack value is null and branches accordingly. These opcodes are commonly used before dereference operations and in fast null-guard paths.

  • {link:ifnull}: branches if the value is null.
  • {link:ifnonnull}: branches if the value is not null.
Operands:
  • label: target label encoded as a signed 16-bit offset relative to the current opcode.
Format:
  • ifnull <label>
  • ifnonnull <label>
Examples:
  • ifnull null_path (guard before getfield / invokevirtual)
  • ifnonnull cast_path (branch to checkcast only when non-null)
Stack Effect:
  • Before: ..., value (reference)
  • After: ... (consumes one category-1 reference value)
Verification:
  • Verification requires that the operand stack can accept one category-1 reference value at the point of this instruction.
  • Branch target and frame-type consistency are verified at class loading/linking time.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions. Invalid branch targets are rejected during verification.
