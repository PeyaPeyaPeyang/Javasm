Store Static Field Instruction

{link:putstatic} - 0xb3 (179)

Purpose

Stores a value into a static field of a class. The value is popped from the operand stack and assigned to the specified static field.

This corresponds to JVM opcode putstatic. The symbolic field reference is resolved before write, and class/interface initialization is triggered if required.

Operands:
  • FieldDescriptor: Specifies the class, field name, and type descriptor of the static field to store the value into. The format is Owner->FieldName:TypeDescriptor. The Owner does not include the L and ; typically used in type descriptors.
Format:

putstatic <FieldDescriptor>

Examples:
  • putstatic java/lang/System->out:Ljava/io/PrintStream;
  • putstatic java/lang/Math->PI:D
TypeDescriptor:

A TypeDescriptor specifies the type of the field. It must represent a valid Java type:

  • I: int
  • J: long
  • F: float
  • D: double
  • Z: boolean
  • B: byte
  • C: char
  • S: short
  • L<classname>;: A reference type (e.g., Ljava/lang/String;)
  • [: An array type (e.g., [I for int[])
Stack Effect:
  • Before: ..., value (category-1 or category-2 depending on field descriptor)
  • After: ... (consumes the field value)
Verification:
  • Verification checks that the operand stack can provide a value of the field type at the point of this instruction.
  • If type resolution fails, the corresponding linkage error is thrown.
Notes:
  • Verifier type checks require stack value compatibility with the field descriptor.
  • For reference assignment rules, normal JVM assignment compatibility applies.
Runtime Exceptions and Errors:
  • NullPointerException: If the field is an instance field and the object reference is null.
  • IncompatibleClassChangeError: If the resolved field is not static.
  • NoSuchFieldError: If the field cannot be found.
  • IllegalAccessError: If the field is not accessible from the current context.
  • NoClassDefFoundError: If the class containing the field cannot be found.
