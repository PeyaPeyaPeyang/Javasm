Create new reference array

{link:anewarray} - 0xbd (189)

Purpose

Creates a new array of references with component type specified by a TypeDescriptor. The length of the array is popped from the stack.

The symbolic type reference is resolved from the run-time constant pool and must denote a reference component type (class, interface, or array type).

Operands:
  • TypeDescriptor: a constant pool index referencing the class or interface type descriptor. Must represent a reference type (e.g., Ljava/lang/String;), primitive types are not allowed.
Format: anewarray <TypeDescriptor>
Examples:
  • anewarray Ljava/lang/String;
  • anewarray Ljava/util/List;
Stack Effect:
  • Before: ..., count
  • After: ..., arrayref (a new reference array)
Verification:
  • Verification requires a reference-compatible input and guarantees a reference result for successor frames.
  • If type resolution fails, the corresponding linkage error is thrown.
Runtime Exceptions and Errors:
  • NegativeArraySizeException: if the length of the array is negative.
  • OutOfMemoryError: if the Java Virtual Machine cannot allocate the array because it is out of memory.
  • Linkage errors (such as NoClassDefFoundError, IllegalAccessError, or IncompatibleClassChangeError): if the symbolic type reference cannot be resolved or if the resolved type is not accessible.
