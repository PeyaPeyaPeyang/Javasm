Shift Right Instructions

{link:ishr} - 0x7a (122)
{link:lshr} - 0x7b (123)

Purpose

Shifts a value to the right arithmetically (sign-propagating) by a specified number of bits.
The shift amount is masked: only the lowest 5 bits are used for ishr, and the lowest 6 bits for lshr.

  • ishr: arithmetic right shift of an int by an int amount
  • lshr: arithmetic right shift of a long by an int amount
Operands: None
Format: {instruction}
Examples:
  • ishr
  • lshr
Stack Effect:
  • ishr
    Before: ..., value (int), shift (int) (both category-1)
    After: ..., result (int) (category-1)
  • lshr
    Before: ..., value (long), shift (int) (value is category-2)
    After: ..., result (long) (category-2)
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values at the point of this instruction.
Notes:
  • These instructions perform an arithmetic right shift, which means that the sign bit (most significant bit) is replicated to fill the vacated bits on the left. This preserves the sign of the value for signed types.
  • The shift amount is masked to prevent shifts greater than the bit width of the value. For ishr, only the lowest 5 bits of the shift amount are used (i.e., shift amounts are effectively modulo 32). For lshr, only the lowest 6 bits are used (i.e., shift amounts are effectively modulo 64).
  • Using a shift amount that is greater than or equal to the bit width of the value will not cause an error but will result in a shift of zero due to the masking behaviour.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
