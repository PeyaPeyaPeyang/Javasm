Stack Popping Instructions

{link:pop} - 0x57 (87)
{link:pop2} - 0x58 (88)

Purpose

Removes one or two values from the top of the operand stack.

Operands: None
Format: {instruction}
Stack Effect:
  • pop
    Before: ..., value
    After: ...
    (Removes one category-1 value: int, float, reference, etc.)
  • pop2
    Two valid use cases:
    - Case 1 (two category-1 values)
    Before: ..., value2, value1
    After: ...

    - Case 2 (one category-2 value)
    Before: ..., value (long or double)
    After: ...
Examples:
  • pop (e.g. discard an unused int)
  • pop2 (e.g. discard a double or two ints)
Notes:
  • pop is valid only when the top stack value is category-1.
  • pop2 is valid only for the two forms shown above.
  • Invalid category/shape usage is rejected by bytecode verification (commonly surfaced as VerifyError during loading/linking).
  • These instructions do not resolve constant-pool entries and have no instruction-specific runtime exception.
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values at the point of this instruction.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
