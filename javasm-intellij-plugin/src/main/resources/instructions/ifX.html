Conditional Branch Instructions

{link:ifeq} - 0x99 (153)
{link:ifne} - 0x9a (154)
{link:iflt} - 0x9b (155)
{link:ifge} - 0x9c (156)
{link:ifgt} - 0x9d (157)
{link:ifle} - 0x9e (158)

Purpose

Compares a single int value on the operand stack against zero and conditionally branches. These opcodes are the canonical primitive predicates for integer-based control flow.

  • {link:ifeq}: branches if value == 0.
  • {link:ifne}: branches if value != 0.
  • {link:iflt}: branches if value < 0.
  • {link:ifge}: branches if value >= 0.
  • {link:ifgt}: branches if value > 0.
  • {link:ifle}: branches if value <= 0.
Operands:
  • label: target label encoded as a signed 16-bit offset relative to the current opcode.
Format:
  • ifeq <label>
  • ifne <label>
  • iflt <label>
  • ifge <label>
  • ifgt <label>
  • ifle <label>
Examples:
  • ifeq zero_label
  • ifne non_zero_label
  • iflt negative_label
  • ifge non_negative_label
  • ifgt positive_label
  • ifle non_positive_label
  • lcmp ; ifgt greater_label (typical two-stage compare for long values)
Stack Effect:
  • Before: ..., value (int)
  • After: ... (consumes one category-1 int value)
Verification:
  • Verification requires that the operand stack can accept one category-1 int value at the point of this instruction.
  • No constant-pool resolution is performed by these instructions.
  • Branch target and frame-type consistency are verified at class loading/linking time.
Notes:
  • Comparisons are signed; these opcodes do not provide unsigned comparison semantics.
  • The {link:if_icmpX} instructions compare two int values on the stack, while these instructions compare a single int value against zero. Choose the appropriate instruction based on the context.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
