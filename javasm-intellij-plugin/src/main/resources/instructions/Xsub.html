Subtraction Instructions

{link:isub} - 0x64 (100)
{link:lsub} - 0x65 (101)
{link:fsub} - 0x66 (102)
{link:dsub} - 0x67 (103)

Purpose

Subtracts the top operand from the second-top operand on the operand stack and pushes the result back onto the stack. The operation performed depends on the instruction variant.

  • isub: subtracts two int values
  • lsub: subtracts two long values
  • fsub: subtracts two float values
  • dsub: subtracts two double values
Operands: None
Format: {instruction}
Examples:
  • isub
  • dsub
Stack Effect:
  • isub
    Before: ..., value1 (int), value2 (int) (category-1)
    After: ..., result (int) (category-1)
    Operation: value1 - value2
  • lsub
    Before: ..., value1 (long), value2 (long) (category-2)
    After: ..., result (long) (category-2)
    Operation: value1 - value2
  • fsub
    Before: ..., value1 (float), value2 (float) (category-1)
    After: ..., result (float) (category-1)
    Operation: value1 - value2
  • dsub
    Before: ..., value1 (double), value2 (double) (category-2)
    After: ..., result (double) (category-2)
    Operation: value1 - value2
Notes:
  • Integer subtraction (isub, lsub) uses two's complement arithmetic. Overflow is ignored and results wrap around silently.
  • Floating-point subtraction (fsub, dsub) follows IEEE 754 semantics:
    • If either operand is NaN, the result is NaN.
    • Subtraction involving infinities follows IEEE 754 rules.
    • Signed zero is preserved where applicable (e.g., +0.0 - +0.0 = +0.0).
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values for the instruction type at the point of this instruction.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
