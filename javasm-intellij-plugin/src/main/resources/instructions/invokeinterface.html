Invoke Interface Method

{link:invokeinterface} - 0xb9 (185)

Purpose

Invokes an interface instance method using interface dispatch semantics. The receiver object is taken from the operand stack and must be non-null.

Operands:
  • methodref (2 bytes): Index into the constant pool, pointing to an InterfaceMethodref
  • count (1 byte): Number of argument slots, including receiver this; category-2 parameters (long/double) count as two slots
  • zero (1 byte): Must be 0 (reserved for future use)
Format:
invokeinterface <methodref> <count> 0
Methodref format:

The method reference format specifies the target method as follows:

<classname>-><methodname>(<argument_descriptor>)<return_descriptor>
  • <classname>: Fully qualified class or interface name in JVM internal form (slash-separated), e.g., java/lang/String
  • <methodname>: Method name, e.g., substring, size
  • <argument_descriptor>: JVM method argument descriptor, e.g., I (int), Ljava/lang/String; (String object)
  • <return_descriptor>: JVM return type descriptor, e.g., V (void), I (int), Ljava/lang/Object; (Object)
Examples:
  • invokeinterface java/lang/String->awdw$1_35(Ljava/lang/String;)Ljava/lang/String;
  • invokeinterface my/pkg/MyInterface->doSomething(I)V
  • invokeinterface java/util/List->size()I
Stack Effect:

Before: ..., objectref, arg1, arg2, ..., argN
After: ..., returnValue? (if return type is non-void)

Pops receiver plus all descriptor arguments. Pushes return value according to the method descriptor (category-2 for long/double, category-1 otherwise).

Notes:
  • This instruction explicitly encodes argument-slot count, unlike invokevirtual.
  • The symbolic interface method is resolved, then dispatch selects an implementation for the runtime class.
  • Default methods may participate in method selection according to JVM interface resolution rules.
Verification:
  • Verification checks that the method reference is valid, the operand stack has a compatible object reference and enough argument slots, and that the resolved method is an interface method.
  • If type resolution fails, the corresponding linkage error is thrown.
  • At runtime, the JVM performs interface method dispatch based on the actual class of the receiver object, following the JVM's method resolution and selection rules for interfaces.
Runtime Exceptions and Errors:
  • NullPointerException: if the receiver object reference is null.
  • IncompatibleClassChangeError: if the resolved method is not an interface method.
  • AbstractMethodError: if the resolved method is abstract and the actual class of the receiver does not provide an implementation.
  • LinkageError: if the symbolic method reference cannot be resolved or if the resolved method is not accessible.
  • WrongMethodTypeException: if the actual argument types do not match the method descriptor at runtime.
