Addition Instructions

{link:iadd} - 0x60 (96)
{link:ladd} - 0x61 (97)
{link:fadd} - 0x62 (98)
{link:dadd} - 0x63 (99)

Purpose

Adds the top two operands on the operand stack and pushes the result, respecting the instruction's type.

  • {link:iadd} and {link:ladd} use two's-complement arithmetic with wraparound on overflow.
  • {link:fadd} and {link:dadd} follow IEEE 754 semantics (including NaN propagation and signed zero behaviour).
Operands: None
Format: {instruction}
Examples:
  • iadd
  • dadd
Stack Effect:
  • {link:iadd}, {link:fadd}
    Before: ..., value1, value2 (category-1)
    After: ..., result (category-1)
  • {link:ladd}, {link:dadd}
    Before: ..., value1, value2 (category-2 values)
    After: ..., result (category-2)
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values at the point of this instruction.
  • No constant-pool resolution is performed by these instructions.
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
