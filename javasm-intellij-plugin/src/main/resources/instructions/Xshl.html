Shift Left Instructions

{link:ishl} - 0x78 (120)
{link:lshl} - 0x79 (121)

Purpose

Shifts a value to the left logically by a specified number of bits.
The shift amount is masked: only the lowest 5 bits are used for ishl, and the lowest 6 bits for lshl.

  • ishl: logically left-shifts an int by an int amount
  • lshl: logically left-shifts a long by an int amount
Operands: None
Format: {instruction}
Examples:
  • ishl
  • lshl
Stack Effect:
  • ishl
    Before: ..., value (int), shift (int) (both category-1)
    After: ..., result (int) (category-1)
  • lshl
    Before: ..., value (long), shift (int) (value is category-2)
    After: ..., result (long) (category-2)
Verification:
  • Verification requires that the operand stack can accept the appropriate category of values at the point of this instruction.
Notes:
  • These instructions perform a logical left shift, meaning that zero bits are shifted in on the right. The sign bit is not treated specially.
  • The shift amount is masked to prevent shifts greater than the bit width of the value. For ishl, only the lowest 5 bits of the shift amount are used (i.e., shift amounts are effectively modulo 32). For lshl, only the lowest 6 bits are used (i.e., shift amounts are effectively modulo 64).
Runtime Exceptions and Errors:

These instructions do not throw any instruction-specific runtime exceptions.
