Multidimensional Array Creation Instruction

{link:multianewarray} - 0xc5 (197)

Purpose

Creates a new multidimensional array of a specified type. The sizes for the dimensions to be initialized are taken from the operand stack. Only the specified number of leading dimensions are allocated.

Operands:
  • indexbyte1, indexbyte2: Index into the runtime constant pool of the current class, referencing a symbolic reference to an array type.
  • dimensions: An unsigned byte indicating the number of dimensions to initialize.
Format:

multianewarray <indexbyte1> <indexbyte2> <dimensions>

Examples:
  • multianewarray [[I 2
  • multianewarray [[[Ljava/lang/String; 2
TypeDescriptor:

The referenced constant pool entry must be a symbolic reference to an array type descriptor. The descriptor uses the following encoding:

  • [: Indicates an array dimension
  • Primitive types: I, J, F, D, Z, B, C, S
  • L<classname>;: Reference type
Stack Effect:
  • Before: ..., count1, count2, ..., countN (all int, category-1)
  • After: ..., arrayref (category-1)

The counts are taken in order, where count1 corresponds to the outermost dimension.

Notes:
  • The value of dimensions must be less than or equal to the number of dimensions in the array type descriptor.
  • Only the first dimensions dimensions are allocated. Remaining dimensions are left uninitialized, meaning their elements are null references.
  • Multidimensional arrays in the JVM are implemented as arrays of arrays.
Runtime Exceptions and Errors:
  • NegativeArraySizeException: If any dimension size is negative.
  • ClassNotFoundException (wrapped in NoClassDefFoundError): If the array type cannot be resolved.
  • IllegalAccessError: If the referenced class or interface is not accessible.
