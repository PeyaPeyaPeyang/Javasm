This inspection analyzes all instructions in the code and reports instructions that are unreachable at runtime.

An instruction is considered unreachable if the control flow cannot reach it due to preceding unconditional control transfer, such as jumps or method exits.

The following instructions are considered to cause unconditional control flow termination:

  • goto
  • return
  • ireturn, lreturn, freturn, dreturn, areturn
  • athrow
  • tableswitch and lookupswitch (when all branches jump away)

Note: jsr (jump to subroutine) is not considered to terminate control flow, as execution resumes after the subroutine completes.

Instructions that follow such terminating instructions and are not the target of any jump or exception handler are flagged with a warning.
