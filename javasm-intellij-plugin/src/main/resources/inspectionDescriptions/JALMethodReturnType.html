This inspection targets return instructions, including typed variants such as ireturn, lreturn, freturn, dreturn, areturn, and the void return.

It checks whether the type of each return instruction matches the return type specified in the method descriptor. If the types are inconsistent, an error is reported.

For example:
  • Using ireturn in a method declared to return double is invalid.
  • Using return in a method that returns int is also invalid.

Ensuring consistency between the method's declared return type and its actual return instructions is essential for valid bytecode.
