例外のスロー

{link:athrow} - 0xbf (191)

目的

オペランド・スタック先頭の例外オブジェクトをスローします。オブジェクトは java.lang.Throwable のサブクラス参照である必要があります。

athrow 実行後，現在命令列の通常実行は継続しません。JVM は現在フレームで例外ハンドラ探索を開始し，必要に応じてフレームをアンロードして呼び出し元に伝播します。例外がキャッチされない場合は，スレッドが終了します。

オペランド: なし
形式: athrow
例:
  • athrow
スタック効果:
  • Before: ..., throwable-instance
  • After: (no values)（例外のスローによりメソッド実行が中断される）
検証:
  • この命令位置で，オペランド・スタックが参照値を受け入れ可能であることが必要です。
実行時例外とエラー:
  • NullPointerException: スタック先頭参照が null の場合。
  • ClassCastException: スタック先頭参照が java.lang.Throwable のインスタンスでない場合。
  • NoClassDefFoundError: スタック先頭参照のクラスが見つからない場合。
  • IncompatibleClassChangeError: スタック先頭参照のクラスが java.lang.Throwable でない場合。
