インターフェース・メソッドの呼び出し

{link:invokeinterface} - 0xb9 (185)

目的

インターフェースディスパッチ規則でインスタンスメソッドを呼び出します。 受け手オブジェクト参照はオペランド・スタックから取得され，null であってはなりません。

オペランド:
  • methodref（2 バイト）: InterfaceMethodref を指す定数プールインデックス
  • count（1 バイト）: 受け手 this を含む引数スロット数。 カテゴリ 2 引数（long/double）は 2 スロットとして数えます
  • zero（1 バイト）: 0 固定（将来拡張用予約）
形式:
invokeinterface <methodref> <count> 0
Methodref format:

メソッド参照形式は，対象メソッドを次のように指定します。

<classname>-><methodname>(<argument_descriptor>)<return_descriptor>
  • <classname>: JVM 内部形式（スラッシュ区切り）の完全修飾クラス/インターフェース名。例: java/lang/String
  • <methodname>: メソッド名。例: substring, size
  • <argument_descriptor>: JVM メソッド引数記述子。例: I，Ljava/lang/String;
  • <return_descriptor>: JVM 戻り値型記述子。例: V，I，Ljava/lang/Object;
例:
  • invokeinterface java/lang/String->awdw$1_35(Ljava/lang/String;)Ljava/lang/String;
  • invokeinterface my/pkg/MyInterface->doSomething(I)V
  • invokeinterface java/util/List->size()I
スタック効果:

Before: ..., objectref, arg1, arg2, ..., argN
After: ..., returnValue?（戻り型が void でない場合）

受け手参照と記述子引数をすべてポップし，メソッド記述子に従って戻り値をプッシュします （long/double は カテゴリ 2，それ以外は カテゴリ 1）。

備考:
  • この命令は invokevirtual と異なり，引数スロット数を明示的にエンコードします。
  • シンボリックインターフェースメソッドを解決後，実行時クラスに対して実装選択ディスパッチを行います。
  • JVM のインターフェース解決規則に従い，default メソッドが選択に関与する場合があります。
検証:
  • 検証では，メソッド参照の妥当性，互換な受け手参照，十分な引数スロット，および解決先がインターフェースメソッドであることを確認します。
  • 型解決に失敗した場合は，対応するリンケージエラーが発生します。
  • 実行時には，受け手実クラスに基づき JVM のインターフェース解決・選択規則でディスパッチされます。
実行時例外とエラー:
  • NullPointerException: 受け手オブジェクト参照が null の場合。
  • IncompatibleClassChangeError: 解決先メソッドがインターフェースメソッドでない場合。
  • AbstractMethodError: 解決先メソッドが abstract で，受け手実クラスが実装を提供しない場合。
  • LinkageError: シンボリックメソッド参照を解決できない場合，または解決先メソッドにアクセスできない場合。
  • WrongMethodTypeException: 実引数型が実行時にメソッド記述子と一致しない場合。
