オブジェクトの新規生成

{link:new} - 0xbb (187)

目的

指定クラスの新規オブジェクト用メモリを確保します。この時点ではオブジェクトは初期化されません。 コンストラクタは invokespecial 命令で明示的に呼び出す必要があります。

プッシュされる値は 未初期化 オブジェクト参照であり，対応するコンストラクタ呼び出しが完了するまで検証器が追跡します。

オペランド:
  • ClassRef: 非配列・非インターフェースクラスを示す CONSTANT_Class_info への定数プールインデックス。
形式:

new <ClassRef>

例:
  • new java/lang/String の後に invokespecial java/lang/String-><init>()V
  • new java/util/ArrayList の後に対応コンストラクタを呼び出し
クラス参照の制約:
  • 解決先はクラス型である必要があり，配列型およびインターフェースは不可です。
  • クラスアクセス検査およびロード/リンク規則は解決時に適用されます。
スタック効果:

新規確保されたオブジェクトへの未初期化参照がオペランド・スタックへ追加されます。

  • Before: ...
  • After: ..., uninitialized-objectref（カテゴリ 1 参照）
検証:
  • この命令位置で，オペランド・スタックが参照値を受け入れ可能であることが必要です。
  • 未初期化参照について，コンストラクタ呼び出しパターンの妥当性が検証されます。
  • 型解決に失敗した場合は，対応するリンケージエラーが発生します。
備考:
  • 検証器は，参照が通常利用される前に正しく初期化フローを通ることを強制します。
  • オブジェクト生成とコンストラクタ実行は，意味論上分離されています。
実行時例外とエラー:
  • OutOfMemoryError: JVM がメモリ不足でオブジェクトを確保できない場合。
  • LinkageError: シンボリッククラス参照を解決できない場合，または解決先クラスにアクセスできない場合。
  • InstantiationError: 解決先クラスが抽象クラスまたはインターフェースである場合。
  • IncompatibleClassChangeError: 解決先がクラスでない場合（配列型やインターフェースなど），またはその他の制約違反がある場合。
