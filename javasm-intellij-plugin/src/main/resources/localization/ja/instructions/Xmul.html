乗算

{link:imul} - 0x68 (104)
{link:lmul} - 0x69 (105)
{link:fmul} - 0x6a (106)
{link:dmul} - 0x6b (107)

目的

オペランド・スタック先頭の 2 値を乗算し，結果をプッシュします。演算は命令の型に従って行われます。

  • {link:imul}/{link:lmul}: 2 の補数乗算（オーバーフロー時はラップアラウンド）。
  • {link:fmul}/{link:dmul}: IEEE 754 乗算（NaN 伝播，符号付きゼロ，無限大を含む）。
オペランド: なし
形式: {instruction}
例:
  • imul
  • dmul
スタック効果:
  • {link:imul}, {link:fmul}
    Before: ..., value1, value2（カテゴリ 1）
    After: ..., result（カテゴリ 1）
  • {link:lmul}, {link:dmul}
    Before: ..., value1, value2（カテゴリ 2 値）
    After: ..., result（カテゴリ 2）
備考:
  • {link:imul} と {link:lmul} は 2 の補数乗算を行います。数学的結果が型の範囲を超える場合，imul は 2^32，lmul は 2^64 を法として循環します。
  • {link:fmul} と {link:dmul} は IEEE 754 規則に従い，NaN・無限大・符号付きゼロを処理します。
  • これらの命令は定数プール解決を行わず，命令固有の実行時例外はありません。
検証:
  • この命令位置で，オペランド・スタックが命令に対応したカテゴリの値を受け入れ可能であることが必要です（imul/fmul は カテゴリ 1 を 2 つ，lmul/dmul は カテゴリ 2 を 2 つ）。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。
