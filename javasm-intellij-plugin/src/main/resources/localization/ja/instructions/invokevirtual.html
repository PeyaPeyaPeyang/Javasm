インスタンス・メソッドの呼び出し

{link:invokevirtual} - 0xb6 (182)

目的

指定オブジェクト参照に対してインスタンス（virtual）メソッドを実行します。実際に呼び出されるメソッドは オブジェクトの実行時型に基づいて決定され，多相性を実現する 動的ディスパッチ が行われます。 クラス階層でオーバーライドされたメソッド呼び出しに一般的に使われます。

invokespecial（動的ディスパッチを行わない）とは異なり，invokevirtual は 受け手オブジェクトの実行時クラスのメソッド表（vtable）を参照します。 これにより，宣言型がスーパークラスやインターフェースでも，最も具体的な実装が選択されます。

オペランド:
  • methodref（2 バイト）: CONSTANT_Methodref_info を指す定数プールインデックス。 そこにはクラス情報とメソッド名・記述子が含まれます。
形式:
invokevirtual <methodref>
Methodref format:

シンボリック参照は実行時に解決され，対象メソッドを特定します。

<classname>-><methodname>(<argument_descriptor>)<return_descriptor>
  • <classname>: JVM 内部形式の完全修飾クラス名（例: java/lang/String）
  • <methodname>: メソッド名（例: equals, length）
  • <argument_descriptor>: JVM 記述子で表した引数型（例: I, Ljava/lang/Object;）
  • <return_descriptor>: 戻り値型記述子（例: V は void, Ljava/lang/String; は String）
例:
  • invokevirtual java/lang/String->length()I
  • invokevirtual java/util/List->get(I)Ljava/lang/Object;
  • invokevirtual java/lang/Object->toString()Ljava/lang/String;
スタック効果:

オペランド・スタックには，オブジェクト参照とメソッド引数が左から右の順で積まれている必要があります。

Before: ..., objectref, arg1, arg2, ..., argN
After: ..., returnValue?

オブジェクト参照と全引数を消費します。メソッドが値を返す場合はその値がプッシュされます。 戻り型が void の場合は何もプッシュされません。 戻り値カテゴリは記述子規則に従います（long/double は カテゴリ 2）。

Dispatch Insight:
  • 解決フェーズでシンボリックメソッド参照とアクセス制約を検証します。
  • その後，動的ディスパッチにより受け手クラス階層内の最も具体的な実装が選択されます。
  • この 2 段階モデルは，多相挙動と遅延失敗の理解に重要です。
Tutorial Pattern:
  • 安全な典型シーケンスは，instanceof → 分岐 → checkcast → invokevirtual です。
  • バイトコード変換時は，受け手位置と記述子スロット計算を厳密に保持してください。
備考:
  • 解決先は，virtual 呼び出しに適したインスタンスメソッドである必要があります。
  • <init> と <clinit> は invokevirtual では呼び出しません。
  • コンストラクタ，private メソッド，明示的 super 呼び出しには invokespecial を使用します。
検証:
  • 検証では，オペランド・スタックに参照互換なオブジェクトと，メソッド記述子に一致する引数があることを確認します。
  • シンボリック参照を解決できない場合，または解決先メソッドにアクセスできない場合はリンケージエラーとなります。
実行時例外とエラー:
  • NullPointerException: オブジェクト参照が null の場合。
  • IncompatibleClassChangeError: 解決先メソッドが static の場合，または解決先クラスがインターフェースの場合。
  • AbstractMethodError: 解決先メソッドが abstract で，受け手実クラスに実装がない場合。
  • LinkageError: シンボリックメソッド参照を解決できない場合，または解決先メソッドにアクセスできない場合。
  • WrongMethodTypeException: 実引数型が実行時にメソッド記述子と一致しない場合。
