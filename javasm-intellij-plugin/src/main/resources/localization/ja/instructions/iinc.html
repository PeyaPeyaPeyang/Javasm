整数のインクリメント

{link:iinc} - 0x84 (132)

目的

int 型ローカル変数に対して，その場で加算（または減算）を行います。オペランド・スタックを使わずに ループ・カウンタや局所的なカウンタを更新する場合によく用いられます。

ローカル変数インデックスと増分値は，いずれもオペランドとして指定します。増分定数は符号付き値のため， 正負どちらの変化量も指定できます。

オペランド:
  • index: 符号なし byte（0〜255）- ローカル変数配列のインデックス
  • const: 符号付き byte（-128〜127）- 増分値

いずれかのオペランドが 1 バイト範囲を超える場合，wide 接頭辞が必要です。 この場合，オペランドは 2 バイトへ拡張されます。

  • wide iinc では，index は符号なし 16 ビット，const は符号付き 16 ビットです。
  • 加算は 32 ビット 2 の補数 int の算術規則に従います。
形式:
  • iinc <index>, <const>
  • wide iinc <index>, <const>（拡張オペランド使用時）
意味論:

疑似コードでは次のように表現されます。 

local_variables[index] = local_variables[index] + const;

オーバーフローは 2^32 の剰余で折り返し，JVM の通常の int 算術と同一です。

例:
  • iinc 2 1 - インデックス 2 のローカル変数を 1 増加
  • iinc 5, -3 - インデックス 5 のローカル変数を 3 減少
  • wide iinc 258, 1000 - インデックス 258 を 1000 増加（wide が必要）
スタック効果:

なし - この命令はオペランド・スタックを読み書きしません。

備考:
  • int 型のローカル変数にのみ適用できます。long など他の型に使うことはできません。
  • iinc は他の算術命令と異なり，スタックへの push/pop を介さずローカル変数スロットを直接更新するため，単純なカウンタ更新では効率的です。
  • コンパイラは i++ や i += constant のような構文でよくこの命令を生成します。
  • すべての算術処理がスタック経由だと仮定するバイトコード計測ツールでは，期待どおり動作しない場合があります。
検証:
  • 検証では，指定インデックスのローカル変数が int 型であり，増分定数が有効な符号付き byte（wide 使用時は short）であることが必要です。
実行時例外とエラー:

この命令固有の実行時例外は発生しません。
