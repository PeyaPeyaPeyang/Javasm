剰余

{link:irem} - 0x70 (112)
{link:lrem} - 0x71 (113)
{link:frem} - 0x72 (114)
{link:drem} - 0x73 (115)

目的

2 番目の値を先頭の値で割った剰余を計算してプッシュします。整数型で除数が 0 の場合は ArithmeticException が発生し，浮動小数点型は IEEE 754 の規則に従います。

  • {link:irem}/{link:lrem}: 結果の符号は被除数に従い，0 除算は例外です。
  • {link:frem}/{link:drem}: IEEE 754 の剰余規則（NaN 伝播を含む）。
オペランド: なし
形式: {instruction}
例:
  • irem
  • drem
スタック効果:
  • {link:irem}, {link:frem}
    Before: ..., value1, value2（カテゴリ 1，value1 は被除数）
    After: ..., remainder（カテゴリ 1）
  • {link:lrem}, {link:drem}
    Before: ..., value1, value2（カテゴリ 2 値，value1 は被除数）
    After: ..., remainder（カテゴリ 2）
検証:
  • この命令位置で，オペランド・スタックが命令に対応したカテゴリの値を受け入れ可能であることが必要です。
備考:
  • frem と drem は IEEE 754 の規則に従うため，0 除算は例外ではなく特定の値（Infinity または NaN）になります。
実行時例外:
  • ArithmeticException は irem または lrem で除数（先頭オペランド）が 0 の場合に発生します。
