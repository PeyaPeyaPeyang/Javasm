整数の比較

{link:if_icmpeq} - 0x9f (159)
{link:if_icmpne} - 0xa0 (160)
{link:if_icmplt} - 0xa1 (161)
{link:if_icmpge} - 0xa2 (162)
{link:if_icmpgt} - 0xa3 (163)
{link:if_icmple} - 0xa4 (164)

目的

オペランド・スタック上の 2 つの int 値を符号付き整数比較し，条件に応じて分岐します。

  • {link:if_icmpeq}: value1 == value2 のとき分岐。
  • {link:if_icmpne}: value1 != value2 のとき分岐。
  • {link:if_icmplt}: value1 < value2 のとき分岐。
  • {link:if_icmpge}: value1 >= value2 のとき分岐。
  • {link:if_icmpgt}: value1 > value2 のとき分岐。
  • {link:if_icmple}: value1 <= value2 のとき分岐。

ここで value2 はスタック先頭，value1 はその直下の値です。

オペランド:
  • label: 現在命令位置を基準とする符号付き 16 ビット相対オフセットで表される分岐先ラベル。
形式:
  • if_icmpeq <label>
  • if_icmpne <label>
  • if_icmplt <label>
  • if_icmpge <label>
  • if_icmpgt <label>
  • if_icmple <label>
例:
  • if_icmpeq equal_label
  • if_icmpne not_equal_label
  • if_icmplt less_than_label
  • if_icmpge greater_or_equal_label
  • if_icmpgt greater_than_label
  • if_icmple less_or_equal_label
スタック効果:
  • Before: ..., value1 (int), value2 (int)
  • After: ...（カテゴリ 1 の int 値を 2 つ消費）
備考:
  • 比較は符号付きです。これらの命令は符号なし比較には使用できません。
  • 比較対象の片方が 0 である場合は{link:ifX} 命令を使用して，スタック上の単一の int 値を 0 と比較することができます。用途に応じて適切な命令を選択してください。
検証:
  • この命令位置で，オペランド・スタックが カテゴリ 1 の int 値 2 つを受け入れ可能であることが必要です。
  • これらの命令は定数プール解決を行いません。
  • 分岐先妥当性とフレーム型整合性は，クラスのロード/リンク時に検証されます。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。不正な分岐先は検証時に拒否されます。
