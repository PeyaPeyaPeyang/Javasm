目的

オペランド・スタック先頭の値を，インデックス 0〜3 のローカル変数へ格納します。値の型は命令の接頭辞で決まります。

オペランド: なし
形式: {instruction}
例:
  • istore_1
  • astore_0
  • dstore_3
スタック効果:
  • istore_<n>, fstore_<n>, astore_<n>
    Before: ..., value（カテゴリ 1）
    After: ...
  • lstore_<n>, dstore_<n>
    Before: ..., value（カテゴリ 2）
    After: ...
検証:
  • この命令位置で，オペランド・スタックが適切なカテゴリの値を供給できる必要があります。
備考:
  • これらの命令は，明示的なローカル変数インデックスを取る istore，lstore，fstore，dstore，astore の短縮形です。
  • バイトコードサイズを節約するため，インデックス 0〜3 の場合に使用されます。
実行時例外とエラー:
  • ローカル変数配列サイズに対してインデックス（0〜3）が範囲外の場合，IndexOutOfBoundsException。
