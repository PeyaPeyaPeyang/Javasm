モニタ・ロックの取得

{link:monitorenter} - 0xc2 (194)

目的

オペランド・スタック先頭のオブジェクト参照に対するモニタ・ロックを取得します。同期ブロックまたは同期メソッドへのエントリで使用されます。

現在スレッドが既に当該モニタを保持している場合，リエントラント・ロックを取得し，モニタの保持カウンタをインクリメントします。

オペランド: なし
形式: monitorenter
例:
  • monitorenter
スタック効果:
  • Before: ..., objectref
  • After: ...（カテゴリ 1 参照を 1 つ消費）
実行時・並行性に関する備考:
  • モニタ取得は，所有権が利用可能になるまでブロックされる場合があります。
  • monitorenter 成功時には，Java メモリモデルで定義される同期順序が確立されます。
使用上の注意

monitorenter は，取得したモニタロックを正しく解放するために常に monitorexit と対で使用する必要があります。通常は同期ブロック先頭で monitorenter，末尾で monitorexit を使用します。

コンパイラ生成パターンでは，try/finally 相当の形により，通常終了時と例外終了時の双方でロックが解放されます。

検証:
  • この命令位置で，オペランド・スタックが参照値を受け入れ可能であることが必要です。
実行時例外とエラー:
  • NullPointerException: スタック先頭のオブジェクト参照が null の場合。
