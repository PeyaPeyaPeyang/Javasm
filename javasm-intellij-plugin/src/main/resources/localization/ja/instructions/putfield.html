インスタンス・フィールドヘの格納

{link:putfield} - 0xb5 (181)

目的

オブジェクトのインスタンス・フィールドへ値を格納します。オブジェクト参照と値はオペランド・スタックからポップされ， 指定フィールドへ代入されます。

代入前に参照が実行時定数プールから解決されます。

オペランド:
  • FieldDescriptor: 格納対象インスタンス・フィールドのクラス，フィールド名，型記述子を指定します。 形式は Owner->FieldName:TypeDescriptor です。 Owner には型記述子で使う L と ; は含めません。
形式:

putfield <FieldDescriptor>

例:
  • putfield java/lang/String->value:[C
  • putfield java/util/ArrayList->size:I
TypeDescriptor:

TypeDescriptor はフィールド型を示し，有効な Java 型である必要があります。

  • I: int
  • J: long
  • F: float
  • D: double
  • Z: boolean
  • B: byte
  • C: char
  • S: short
  • L<classname>;: 参照型（例: Ljava/lang/String;）
  • [: 配列型（例: [I は int[]）
スタック効果:
  • Before: ..., objectref, value（value は記述子に応じて カテゴリ 1 または カテゴリ 2）
  • After: ...（値とオブジェクト参照を消費）
検証:
  • この命令位置で，オペランド・スタックに参照互換なオブジェクト参照と，フィールド型に適合する値があることが必要です。
  • 型解決に失敗した場合は，対応するリンケージエラーが発生します。
備考:
  • カテゴリ 2 フィールド型（long，double）では，値 2 ワード + objectref 1 ワードのスロット計算になります。
  • 検証器は各制御フロー点で，フィールド記述子に対する値型の代入可能性を検査します。
実行時例外とエラー:
  • NullPointerException: オブジェクト参照が null の場合。
  • IncompatibleClassChangeError: 解決先フィールドがインスタンス・フィールドでない場合（static など）。
  • NoSuchFieldError: フィールド名または型が解決先クラス内のどのフィールドにも一致しない場合。
  • IllegalAccessError: フィールドが現在クラスからアクセス不可の場合（例: 他クラスの private フィールド）。
  • NoClassDefFoundError: フィールドを含むクラスを実行時に見つけられない場合。
