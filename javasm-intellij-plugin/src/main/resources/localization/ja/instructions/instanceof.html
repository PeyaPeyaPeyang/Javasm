インスタンス型の検査

{link:instanceof} - 0xc1 (193)

目的

オペランド・スタック先頭のオブジェクト参照が，指定された参照型のインスタンスかどうかを判定します。真なら 1，偽なら 0 をプッシュします。

対象型は，実行時定数プールのシンボリッククラス参照を解決して取得されます。

オペランド:
  • TypeDescriptor: クラス，インターフェース，または配列型を表す定数プール参照 （例: Ljava/lang/String;, [Ljava/lang/Object;）。 プリミティブ型は対象にできません。
形式: instanceof <TypeDescriptor>
例:
  • instanceof Ljava/lang/String;
  • instanceof Ljava/util/List;
スタック効果:
  • Before: ..., objectref
  • After: ..., result（カテゴリ 1 の int: 0 または 1）
意味論:
  • objectref が null の場合，結果は 0 です（例外はスローされません）。
  • null でない場合，実行時クラスが対象型に代入可能なときに限り結果は 1 です。
  • 配列型が対象の場合も，JVM の通常の配列代入可能性規則が適用されます。
備考:
  • 典型パターン: instanceof T の後に条件分岐し，checkcast T を実行する。
  • 多相コードで値依存のキャストを安全に行う前段として利用します。
検証:
  • 検証では，参照互換の入力が必要であり，後続フレームに対して int 結果が保証されます。
  • 型解決に失敗した場合は，対応するリンケージエラーが発生します。
  • instanceof には ClassCastException 経路はなく，判定失敗時は 0 を返します。
実行時例外とエラー:
  • NoClassDefFoundError，IllegalAccessError，IncompatibleClassChangeError などのリンケージエラー: シンボリック型参照を解決できない場合，または解決後の型にアクセスできない場合。
