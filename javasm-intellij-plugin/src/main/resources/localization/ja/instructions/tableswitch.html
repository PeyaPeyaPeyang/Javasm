範囲ベースの switch

{link:tableswitch} - 0xaa (170)

目的

オペランド・スタック上の int キーに基づいて switch 形式の分岐を行います。キーは指定された下限インデックスから始まる連続範囲と照合され，low からの差分に対応するラベルへ分岐します。キーが範囲外の場合は default ラベルへ分岐します。

オペランド:
  • 命令後に続く最初の 32 ビット項目が 4 バイト境界に揃うように入るパディング用バイト（0x00）。
  • default: 命令の位置を基準とした符号付き 32 ビット分岐オフセット。
  • low, high: 符号付き 32 ビットのキー範囲（両端含む）。
  • jump offsets: high - low + 1 個の符号付き 32 ビットオフセット（キー順に 1 つずつ）。
形式:
tableswitch <low_index> {
  label_0;
  label_1;
  ...
} default <default_label>
例:
tableswitch 1 {
  case_one;
  case_two;
  case_three;
} default default_case;
スタック効果:
  • Before: ..., key（int 値）
  • After: ...（カテゴリ 1 の int を 1 つ消費して分岐）

key が low_index から low_index + number_of_labels - 1 までの連続範囲外であれば，必須の default ラベルへ分岐します。

検証:
  • 計算されるすべての分岐先（default および各 case）は，同一メソッド内の有効な命令境界を指す必要があります。
  • すべての switch 分岐先で，フレーム型がマージ可能である必要があります。
備考:
  • tableswitch はキー範囲が密な場合に最適です。疎なキー集合には通常 lookupswitch が適しています。
実行時例外とエラー:

この命令固有の実行時例外は発生しません。
