静的メソッドの呼び出し

{link:invokestatic} - 0xb8 (184)

目的

オブジェクト参照を消費せずに static メソッドを呼び出します。 呼び出し対象は，解決後のシンボリックメソッド参照から決定されます。

オペランド:
  • methodref（2 バイト）: Methodref を指す定数プールインデックス
形式:
invokestatic <methodref>
Methodref format:

メソッド参照形式は，対象メソッドを次のように指定します。

<classname>-><methodname>(<argument_descriptor>)<return_descriptor>
  • <classname>: JVM 内部形式（スラッシュ区切り）の完全修飾クラス名。例: java/lang/Math
  • <methodname>: メソッド名。例: max, abs
  • <argument_descriptor>: JVM メソッド引数記述子。例: I（int），Ljava/lang/String;（String）
  • <return_descriptor>: JVM 戻り値型記述子。例: V（void），I（int），Ljava/lang/Object;（Object）
例:
  • invokestatic java/lang/Math->max(II)I
  • invokestatic my/pkg/Utils->doStaticThing()V
スタック効果:

Before: ..., arg1, arg2, ..., argN
After: ..., returnValue?（戻り型が void でない場合）

記述子で定義された全引数をポップし，宣言された戻り型に従って戻り値をプッシュします （long/double は カテゴリ 2，それ以外は カテゴリ 1）。

検証:
  • 検証では，引数型とスタック形状がメソッド記述子に一致することを確認します。
  • 呼び出し前に，必要であれば対象クラスまたはインターフェースの初期化が行われます。
備考:
  • 受け手オブジェクトは消費されません。すべての引数は記述子で明示されます。
  • クラス初期化の副作用により，対象メソッド本体実行前にユーザーコードが実行される場合があります。
実行時例外とエラー:
  • LinkageError: シンボリックメソッド参照を解決できない場合，または解決先メソッドにアクセスできない場合。
  • IncompatibleClassChangeError: 解決先メソッドが static でない場合，または呼び出し型に対して無効なメソッドハンドルである場合。
  • IllegalAccessError: 現在クラスから解決先メソッドにアクセスできない場合。
  • WrongMethodTypeException: 実引数型が実行時にメソッド記述子と一致しない場合。
