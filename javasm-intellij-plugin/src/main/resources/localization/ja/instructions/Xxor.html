ビット XOR

{link:ixor} - 0x82 (130)
{link:lxor} - 0x83 (131)

目的

オペランド・スタック先頭の 2 値に対してビット単位の排他的 OR（XOR）演算を行い， 結果をスタックへプッシュします。

  • {link:ixor}: int 値 2 つを対象に演算
  • {link:lxor}: long 値 2 つを対象に演算
オペランド: なし
形式: {instruction}
例:
  • ixor
  • lxor
スタック効果:
  • ixor
    Before: ..., value1 (int), value2 (int)（カテゴリ 1）
    After: ..., result (int)（カテゴリ 1）
    Operation: value1 ^ value2
  • lxor
    Before: ..., value1 (long), value2 (long)（カテゴリ 2）
    After: ..., result (long)（カテゴリ 2）
    Operation: value1 ^ value2
検証:
  • この命令位置で，オペランド・スタックが命令に対応したカテゴリの値を受け入れ可能であることが必要です。
備考:
  • XOR は各ビット位置ごとに独立して実行され，2 つのビットが異なる場合は 1，同じ場合は 0 になります。
  • この演算は値を純粋にビット列として扱うため，符号付き/符号なし解釈には依存しません。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。
