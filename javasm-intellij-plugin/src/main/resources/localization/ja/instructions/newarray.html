配列の新規作成

{link:newarray} - 0xbc (188)

目的

プリミティブ要素型の 1 次元配列を新規作成します。 配列長はオペランド・スタックからポップされ，配列参照がプッシュされます。

オペランド:
  • atype: プリミティブ要素型を選択する符号なし byte:
    • 4: boolean
    • 5: char
    • 6: float
    • 7: double
    • 8: byte
    • 9: short
    • 10: int
    • 11: long
形式:

newarray <atype>

例:
  • newarray 10（int[] を作成）
  • newarray 7（double[] を作成）
型の適用範囲:

newarray はプリミティブ配列専用です。 参照型要素の配列には anewarray，多次元配列には multianewarray を使用します。

スタック効果:

オペランド・スタックには作成する配列サイズ（負でない整数）が必要です。

  • Before: ..., count
  • After: ..., arrayref（カテゴリ 1 参照）
実行時例外とエラー:
  • NegativeArraySizeException: 配列長が負の場合。
  • OutOfMemoryError: JVM がメモリ不足で配列を確保できない場合。
検証:
  • 検証では，配列サイズとして int 値を受け入れ可能であり，かつ atype オペランドが有効であることが必要です。
