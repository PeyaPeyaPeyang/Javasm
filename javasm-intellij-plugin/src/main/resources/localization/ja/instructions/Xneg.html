符号反転

{link:ineg} - 0x74 (116)
{link:lneg} - 0x75 (117)
{link:fneg} - 0x76 (118)
{link:dneg} - 0x77 (119)

目的

オペランド・スタック先頭の値の符号を反転（-1 倍）し，結果をプッシュします。

  • {link:ineg}/{link:lneg}: 2 の補数の符号反転。MIN_VALUE は反転しても同じ値になります。
  • {link:fneg}/{link:dneg}: IEEE 754 の符号反転（+0.0 から -0.0 への変換を含む）。
オペランド: なし
形式: {instruction}
例:
  • ineg
  • dneg
スタック効果:
  • {link:ineg}, {link:fneg}
    Before: ..., value（カテゴリ 1）
    After: ..., -value（カテゴリ 1）
  • {link:lneg}, {link:dneg}
    Before: ..., value（カテゴリ 2）
    After: ..., -value（カテゴリ 2）
備考:
  • {link:ineg} と {link:lneg} は 2 の補数の符号反転を行うため，最小値（例: Integer.MIN_VALUE）を反転するとオーバーフローにより同じ値になります。
  • {link:fneg} と {link:dneg} は IEEE 754 に基づく符号反転です。+0.0 は -0.0 へ，-0.0 は +0.0 へ反転します。
  • これらの命令は定数プール解決を行わず，命令固有の実行時例外はありません。
検証:
  • この命令位置で，オペランド・スタックが命令に対応したカテゴリの値を受け入れ可能であることが必要です。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。
