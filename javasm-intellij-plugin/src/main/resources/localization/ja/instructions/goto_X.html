無条件な分岐

{link:goto} - 0xa7 (167)
{link:goto_w} - 0xc8 (200)

目的

同一メソッド内の分岐先へ無条件に制御を移します。 これらの命令は，ループ，状態遷移，およびコンパイラ生成の制御フロー書き換えの基本要素です。

{link:goto} は符号付き 16 ビットの分岐オフセットを使用します。 {link:goto_w} は符号付き 32 ビットの分岐オフセットを使用します。 いずれも，現在命令の命令位置を基準とした相対オフセットです。

オペランド:
  • label: アセンブラレベルのラベル（バイトコード分岐オフセットへ解決される）。
  • {link:goto}: オフセット範囲 -32768..32767。
  • {link:goto_w}: オフセット範囲 -2147483648..2147483647。
形式:
  • goto <label>
  • goto_w <label>
例:
  • goto loop_start（ループで一般的な後方分岐）
  • goto_w far_target（分岐先が 16 ビット範囲外の場合に使用）
スタック効果:
  • Before: ..., stack
  • After: ..., stack（値の消費・生成なし）
検証と制御フロー規則:
  • 分岐先は，同一メソッド内の有効な命令先頭でなければなりません。
  • すべての制御フロー合流点で，ローカル変数型とオペランド・スタック型が整合している必要があります。
  • 未初期化オブジェクトを含む制御フローは，コンストラクタ初期化規則を満たす必要があります。
備考:
  • 不正な分岐構造は通常，検証時に拒否されます（例: VerifyError）。
  • 分岐距離が必要としない限り，{link:goto} の使用が推奨されます。
検証:
  • すべての分岐先が同一メソッド内の有効な命令オフセットであることを確認します。
  • 制御フロー合流点でスタック型とローカル変数型が整合していることを確認します。
  • 未初期化オブジェクトを伴う制御フローがコンストラクタの初期化規則を満たすことを確認します。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。不正な分岐先は検証時に拒否されます。
