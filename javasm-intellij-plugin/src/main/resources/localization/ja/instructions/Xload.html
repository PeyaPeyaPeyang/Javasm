ローカル変数の読み出し

{link:iload} - 0x15 (21)
{link:lload} - 0x16 (22)
{link:fload} - 0x17 (23)
{link:dload} - 0x18 (24)
{link:aload} - 0x19 (25)

目的

ローカル変数インデックス N の値をオペランド・スタックへロードします。

バイトコード上では，命令に wide 接頭辞が付かない場合，N は符号なし 8 ビットです。 wide 接頭辞付きの場合は符号なし 16 ビットになります。

オペランド:
  • N: ローカル変数インデックス（10 進または 16 進。例: 3, 0x03）
形式: {instruction} N
wide 接頭辞に関する備考:

wide 接頭辞を使用すると，オペランド N は 1 バイトから 2 バイトへ拡張され，255 を超えるローカル変数インデックスを指定できます。

例:
  • iload 2
  • aload 0x0A
  • wide dload 0x0100（インデックス 256）
スタック効果:
  • iload, fload, aload
    Before: ...
    After: ..., value（カテゴリ 1）
  • lload, dload
    Before: ...
    After: ..., value（カテゴリ 2）
    （備考: long と double はスタックスロットを 2 つ使用します）
検証:
  • 検証では，ローカルスロット N の型が選択したロード命令の型と一致している必要があります。
実行時例外とエラー:
  • ローカル変数インデックスが範囲外（例: 負値，またはメソッドのローカル変数配列サイズ超過）の場合，IndexOutOfBoundsException。
