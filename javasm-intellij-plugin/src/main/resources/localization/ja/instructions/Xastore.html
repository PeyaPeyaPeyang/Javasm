配列ストア

{link:iastore} - 0x4f (79)
{link:lastore} - 0x50 (80)
{link:fastore} - 0x51 (81)
{link:dastore} - 0x52 (82)
{link:aastore} - 0x53 (83)
{link:bastore} - 0x54 (84)
{link:castore} - 0x55 (85)
{link:sastore} - 0x56 (86)

目的

配列へ値を格納します。配列参照，インデックス，格納値の順でオペランド・スタックからポップされます。

  • {link:iastore}: int[] に int を格納
  • {link:lastore}: long[] に long を格納
  • {link:fastore}: float[] に float を格納
  • {link:dastore}: double[] に double を格納
  • {link:aastore}: Object[] に参照を格納
  • {link:bastore}: byte[]/boolean[] に byte または boolean を格納
  • {link:castore}: char[] に char を格納
  • {link:sastore}: short[] に short を格納

配列参照が null の場合，またはインデックスが範囲外の場合は例外が発生します。

{link:aastore} では，格納する参照値が，格納先配列の実行時要素型に代入可能である必要があります。

オペランド: なし
形式: {instruction}
例:
  • iastore
  • lastore
  • fastore
  • dastore
  • aastore
  • bastore
  • castore
  • sastore
スタック効果:
  • iastore, fastore, aastore, bastore, castore, sastore
    Before: ..., arrayref, index, value
    After: ...
  • lastore, dastore
    Before: ..., arrayref, index, value（value は カテゴリ 2）
    After: ...
    （備考: long と double はスタックスロットを 2 つ使用します）
検証:
  • この命令位置で，オペランド・スタックが配列参照，インデックス，格納値として適切な型を供給できる必要があります。
  • {link:aastore} では，格納値が参照型であり，配列参照がその参照型配列と互換であることを検証します。
備考:
  • これらの命令は定数プールエントリを解決せず，命令固有のリンケージエラーはありません。
  • カテゴリ 2 値型（long, double）では，オペランド・スタックのスロット計算は「値 2 ワード + arrayref 1 + index 1」となります。
実行時例外とエラー:
  • NullPointerException：配列参照が null の場合。
  • ArrayIndexOutOfBoundsException：インデックスが範囲外の場合。
  • ArrayStoreException：aastore で格納値が配列の実行時要素型に代入可能でない場合。
