参照配列の新規作成

{link:anewarray} - 0xbd (189)

目的

TypeDescriptor で指定された要素型を持つ参照配列を新規作成します。配列長はスタックからポップされます。

シンボリック型参照は実行時定数プールから解決され，参照型の要素型（クラス，インターフェース，配列型）を示している必要があります。

オペランド:
  • TypeDescriptor: クラスまたはインターフェース型記述子を参照する定数プールインデックス。 参照型（例: Ljava/lang/String;）である必要があり，プリミティブ型は使用できません。
形式: anewarray <TypeDescriptor>
例:
  • anewarray Ljava/lang/String;
  • anewarray Ljava/util/List;
スタック効果:
  • Before: ..., count
  • After: ..., arrayref（新しい参照配列）
検証:
  • 検証では，参照互換の入力が必要であり，後続フレームに対して参照結果が保証されます。
  • 型解決に失敗した場合は，対応するリンケージエラーが発生します。
実行時例外とエラー:
  • NegativeArraySizeException: 配列長が負の場合。
  • OutOfMemoryError: JVM がメモリ不足で配列を確保できない場合。
  • NoClassDefFoundError，IllegalAccessError，IncompatibleClassChangeError などのリンケージエラー: シンボリック型参照を解決できない場合，または解決後の型にアクセスできない場合。
