ローカル変数への格納

{link:istore} - 0x36 (54)
{link:lstore} - 0x37 (55)
{link:fstore} - 0x38 (56)
{link:dstore} - 0x39 (57)
{link:astore} - 0x3a (58)

目的

オペランド・スタック上の値を，指定したローカル変数インデックス N に格納します。

バイトコード上では，wide 接頭辞なしの場合 N は符号なし 8 ビットです。 wide 接頭辞付きの場合は符号なし 16 ビットになります。

オペランド:
  • N: ローカル変数インデックス（10 進または 16 進。例: 5, 0x05）
形式: {instruction} N
wide 接頭辞に関する備考:

wide 接頭辞を使用すると，インデックスオペランド N は 1 バイトから 2 バイトに拡張され，大きなローカル変数インデックスを扱えます。

例:
  • istore 3
  • lstore 0x0A
  • wide fstore 0x0100（インデックス 256）
スタック効果:
  • istore, fstore, astore
    Before: ..., value
    After: ...（カテゴリ 1 値をローカル変数 N に格納）
  • lstore, dstore
    Before: ..., value（カテゴリ 2）
    After: ...（値をローカル変数 N に格納）
    （備考: long と double はスタックスロットを 2 つ使用します）
検証:
  • この命令位置で，オペランド・スタックに命令に対応する適切な型の値が存在する必要があります。
  • istore は int，lstore は long，fstore は float，dstore は double，astore は参照型が必要です。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。
