右シフト

{link:ishr} - 0x7a (122)
{link:lshr} - 0x7b (123)

目的

指定ビット数だけ値を算術右シフト（符号拡張あり）します。
シフト量はマスクされ，ishr では下位 5 ビット，lshr では下位 6 ビットのみが使用されます。

  • ishr: int を int 量だけ算術右シフト
  • lshr: long を int 量だけ算術右シフト
オペランド: なし
形式: {instruction}
例:
  • ishr
  • lshr
スタック効果:
  • ishr
    Before: ..., value (int), shift (int)（両方 カテゴリ 1）
    After: ..., result (int)（カテゴリ 1）
  • lshr
    Before: ..., value (long), shift (int)（value は カテゴリ 2）
    After: ..., result (long)（カテゴリ 2）
検証:
  • この命令位置で，オペランド・スタックが命令に対応したカテゴリの値を受け入れ可能であることが必要です。
備考:
  • これらは算術右シフトであり，左側の空いたビットは符号ビット（最上位ビット）で埋められます。これにより符号付き値の符号が保持されます。
  • シフト量は値のビット幅を超えないようにマスクされます。ishr は実質 32 での剰余，lshr は実質 64 での剰余として扱われます。
  • ビット幅以上のシフト量を指定してもエラーにはなりません。マスク後の値で演算されます。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。
