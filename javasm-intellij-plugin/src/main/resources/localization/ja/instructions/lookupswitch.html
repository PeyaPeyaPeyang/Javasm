テーブル検索 switch

{link:lookupswitch} - 0xab (171)

目的

オペランド・スタック先頭の int 値に基づき switch 形式の分岐を行います。キーを match-offset の組と照合し，一致したラベルへ分岐します。一致しない場合は default ラベルへ分岐します。

オペランド:
  • 命令後に続く最初の 32 ビット項目が 4 バイト境界に揃うように入るパディング用バイト（0x00）。
  • default: 命令の位置を基準とした 32 ビット分岐オフセット。
  • npairs: match-offset 組の個数を表す符号付き 32 ビット値（0 以上）。
  • match_i/offset_i: 符号付き 32 ビットのキー値と分岐オフセットの組。
  • match_i キーは厳密な昇順で並んでいる必要があります。
形式:
lookupswitch {
  key_1: label_1;
  key_2: label_2;
  ...
  default: label_default;
}

キーは 10 進または 16 進（例: 0x00）で指定でき，ラベルはシンボリックな分岐先です。

例:
lookupswitch {
  0: case_zero;
  1: case_one;
  default: default_case;
}
スタック効果:
  • Before: ..., key（int 値）
  • After: ...（カテゴリ 1 の int を 1 つ消費して分岐）

キーがどの case にも一致しない場合は，必須の default ラベルへ分岐します。

検証:
  • すべての分岐先は，同一メソッド内の有効な命令境界を指す必要があります。
  • すべての switch 分岐先からの制御フロー合流で，ローカル/スタック型整合性が必要です。
備考:
  • キー集合が疎な場合には lookupswitch を使用します。密なキーは tableswitch で表現されます。
  • キーは必ず昇順でなければなりません。さもなければ，クラスのロード/リンク時に検証エラーになります。
実行時例外とエラー:

この命令固有の実行時例外は発生しません。不正な分岐先は検証時に拒否されます。
