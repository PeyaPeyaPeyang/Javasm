静的フィールドへの格納

{link:putstatic} - 0xb3 (179)

目的

クラスの静的フィールドへ値を格納します。値はオペランド・スタックからポップされ， 指定された静的フィールドへ代入されます。

書き込み前にシンボリックフィールド参照が解決され，必要に応じてクラス/インターフェースの初期化が実行されます。

オペランド:
  • FieldDescriptor: 格納対象静的フィールドのクラス，フィールド名，型記述子を指定します。 形式は Owner->FieldName:TypeDescriptor です。 Owner には型記述子で使う L と ; は含めません。
形式:

putstatic <FieldDescriptor>

例:
  • putstatic java/lang/System->out:Ljava/io/PrintStream;
  • putstatic java/lang/Math->PI:D
TypeDescriptor:

TypeDescriptor はフィールド型を示し，有効な Java 型である必要があります。

  • I: int
  • J: long
  • F: float
  • D: double
  • Z: boolean
  • B: byte
  • C: char
  • S: short
  • L<classname>;: 参照型（例: Ljava/lang/String;）
  • [: 配列型（例: [I は int[]）
スタック効果:
  • Before: ..., value（フィールド記述子に応じて カテゴリ 1 または カテゴリ 2）
  • After: ...（フィールド値を消費）
検証:
  • 検証では，この命令位置でオペランド・スタックがフィールド型の値を供給できることを確認します。
  • 型解決に失敗した場合は，対応するリンケージエラーが発生します。
備考:
  • 検証器の型検査では，スタック値がフィールド記述子と互換であることが必要です。
  • 参照代入規則には通常の JVM 代入互換性が適用されます。
実行時例外とエラー:
  • IncompatibleClassChangeError: 解決先フィールドが static でない場合。
  • NoSuchFieldError: フィールドを見つけられない場合。
  • IllegalAccessError: 現在の文脈からフィールドにアクセスできない場合。
  • NoClassDefFoundError: フィールドを含むクラスを見つけられない場合。
