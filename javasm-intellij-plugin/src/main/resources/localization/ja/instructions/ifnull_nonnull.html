null の判定による分岐

{link:ifnull} - 0xc6 (198)
{link:ifnonnull} - 0xc7 (199)

目的

オペランド・スタック先頭の値が null かどうかを判定し，条件に応じて分岐します。 これらの命令は，高速な null チェックや，参照をデリファレンスする前のガードとしてよく使用されます。

  • {link:ifnull}: 値が null のとき分岐。
  • {link:ifnonnull}: 値が null でないとき分岐。
オペランド:
  • label: 現在命令位置を基準とする符号付き 16 ビット相対オフセットで表される分岐先ラベル。
形式:
  • ifnull <label>
  • ifnonnull <label>
例:
  • ifnull null_path（getfield / invokevirtual 前のガード）
  • ifnonnull cast_path（非 null のときのみ checkcast へ分岐）
スタック効果:
  • Before: ..., value (reference)
  • After: ...（カテゴリ 1 の参照値を 1 つ消費）
検証:
  • この命令位置で，オペランド・スタックが カテゴリ 1 の参照値 1 つを受け入れ可能であることが必要です。
  • 分岐先妥当性とフレーム型整合性は，クラスのロード/リンク時に検証されます。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。不正な分岐先は検証時に拒否されます。
