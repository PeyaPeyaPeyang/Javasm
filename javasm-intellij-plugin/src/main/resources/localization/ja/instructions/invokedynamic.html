動的な呼び出し

{link:invokedynamic} - 0xba (186)

目的

実行時にブートストラップメソッドでリンクされる動的コールサイト呼び出しを実行します。 Java のラムダ変換や言語レベルの動的ディスパッチ機構の基盤です。

他の invoke 命令と異なり，呼び出し先はクラス/メソッドのシンボリック参照だけでは固定されません。 リンケージの結果として CallSite が生成され，その target メソッドハンドルが呼び出されます。

オペランド:
  • バイトコードオペランドは CONSTANT_InvokeDynamic_info を指す定数プールインデックスで，後続に 0 バイトが 2 つ続きます。
  • CONSTANT_InvokeDynamic_info は，ブートストラップメソッド項目と name-and-type 記述子へリンクします。
  • ブートストラップ静的引数は，クラスファイルの BootstrapMethods 属性にエンコードされます。
形式:
invokedynamic <name> <methodDescriptor> [bootstrap = MethodHandle + static args]
詳細:
  • 初回実行時に JVM はコールサイトを解決し，ブートストラップメソッドを呼び出します。
  • ブートストラップメソッドには，lookup 文脈，コールサイト名，メソッド型，静的引数が渡されます。
  • 戻り値は null でない CallSite であり，その target 型はコールサイト記述子と完全一致する必要があります。
  • リンク成功後の実行では，リンク済みコールサイトが再利用されます。
例:
invokedynamic apply (Ljava/lang/Object;)Ljava/lang/Object;
  // bootstrap: java/lang/invoke/LambdaMetafactory.metafactory
スタック効果:

Before: ..., arg1, arg2, ..., argN
After: ..., returnValue
引数と戻り値型は methodDescriptor で定義されます。 カテゴリ 2 値（long/double）は 2 スロットを消費または生成します。

検証:
  • 検証では，コールサイト記述子に対するオペランド・スタック互換性を確認します。
  • リンケージ失敗はリンケージエラー（一般に BootstrapMethodError ラッパー）として報告されます。
  • リンク済みターゲットを実引数型で呼び出せない場合，メソッドハンドル呼び出し型エラーが発生することがあります。
  • この命令に暗黙の objectref はありません。すべての引数は記述子で明示されます。
備考:
  • 呼び出し挙動をブートストラップロジックへ意図的に遅延させる場合に invokedynamic を使用します。
  • Java コンパイルでは，ラムダ式と文字列連結がこの命令の代表的生成元です。
実行時例外とエラー:
  • BootstrapMethodError: ブートストラップメソッドがコールサイトリンクに失敗した場合（例: 例外スロー，null 返却，不整合な target 型を持つ CallSite の返却）。
  • LinkageError: シンボリック参照を解決できない場合，または解決先メソッドにアクセスできない場合。
  • IncompatibleClassChangeError: 解決されたメソッドハンドルが不正，または解決されたコールサイトが不正な場合。
  • WrongMethodTypeException: 実引数型が実行時にコールサイト記述子と一致しない場合。
