キャスト可能性の検査

{link:checkcast} - 0xc0 (192)

目的

オペランド・スタック先頭のオブジェクトが，指定した参照型に適合するかを検査します。 検査に成功した場合，オブジェクトは変更されずそのままスタックに残ります。 失敗した場合は実行時に ClassCastException がスローされます。

対象型は，実行時定数プールのシンボリック参照を解決して取得されます。

この命令は，Java ソースにおける (String) obj のような実行時型検査を実装します。

オペランド:
  • TypeDescriptor: クラス・インターフェース・配列型記述子を指す定数プールインデックス （例: Ljava/lang/String;, [Ljava/lang/Object;）
形式:
checkcast <TypeDescriptor>
スタック効果:
  • Before: ..., objectref
  • After: ..., objectref
意味論:
  • objectref == null の場合，キャストは常に成功します（null は任意の参照型に代入可能）。
  • objectref が null でなく，かつ対象型に代入可能でない場合，ClassCastException がスローされます。
  • オブジェクト参照そのものは変更されず，検査のみが行われます。
例:
  • checkcast Ljava/lang/String; - オブジェクトが String またはそのサブクラスかを検査
  • checkcast Ljava/util/List; - List 互換性を検査
  • checkcast [Ljava/lang/String; - オブジェクトが String[] かを検査
備考:
  • この命令は純粋な型検査であり，オブジェクトの変換や複製は行いません。
  • 参照型間のキャストで，Java コンパイラがよく挿入します。
  • 多相的な処理を実装する際，instanceof と組み合わせて使われます。
  • 対象型であることが既に分かっている場合（例: instanceof の直後）は，checkcast を安全に利用できます。
検証:
  • 検証では，参照互換の入力が必要であり，後続フレームに対して参照結果が保証されます。
  • 型解決に失敗した場合は，対応するリンケージエラーが発生します。
実行時例外とエラー:
  • ClassCastException: オブジェクト参照が null ではなく，対象型と代入互換でない場合。
  • NoClassDefFoundError，IllegalAccessError，IncompatibleClassChangeError などのリンケージエラー: シンボリック参照を解決できない場合，または解決後の型にアクセスできない場合。
