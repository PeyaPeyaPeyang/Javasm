条件分岐

{link:ifeq} - 0x99 (153)
{link:ifne} - 0x9a (154)
{link:iflt} - 0x9b (155)
{link:ifge} - 0x9c (156)
{link:ifgt} - 0x9d (157)
{link:ifle} - 0x9e (158)

目的

オペランド・スタック上の単一の int 値を 0 と比較し，条件に応じて分岐します。 これらの命令は，整数ベース制御フローにおける基本的な述語です。

  • {link:ifeq}: value == 0 のとき分岐。
  • {link:ifne}: value != 0 のとき分岐。
  • {link:iflt}: value < 0 のとき分岐。
  • {link:ifge}: value >= 0 のとき分岐。
  • {link:ifgt}: value > 0 のとき分岐。
  • {link:ifle}: value <= 0 のとき分岐。
オペランド:
  • label: 現在命令位置を基準とする符号付き 16 ビット相対オフセットで表される分岐先ラベル。
形式:
  • ifeq <label>
  • ifne <label>
  • iflt <label>
  • ifge <label>
  • ifgt <label>
  • ifle <label>
例:
  • ifeq zero_label
  • ifne non_zero_label
  • iflt negative_label
  • ifge non_negative_label
  • ifgt positive_label
  • ifle non_positive_label
  • lcmp ; ifgt greater_label（long 比較で典型的な 2 段階比較）
スタック効果:
  • Before: ..., value (int)
  • After: ...（カテゴリ 1 の int 値を 1 つ消費）
検証:
  • この命令位置で，オペランド・スタックが カテゴリ 1 の int 値 1 つを受け入れ可能であることが必要です。
  • これらの命令は定数プール解決を行いません。
  • 分岐先妥当性とフレーム型整合性は，クラスのロード/リンク時に検証されます。
備考:
  • 比較は符号付きです。これらの命令は符号なし比較には使用できません。
  • {link:if_icmpX} 命令は，スタック上の 2 つの int 値を比較しますが，これらの命令はスタック上の単一の int 値を 0 と比較します。用途に応じて適切な命令を選択してください。
実行時例外とエラー:

これらの命令固有の実行時例外は発生しません。
