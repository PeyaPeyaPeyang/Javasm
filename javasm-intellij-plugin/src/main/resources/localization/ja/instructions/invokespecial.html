特殊なメソッドの呼び出し

{link:invokespecial} - 0xb7 (183)

目的

特殊（非 virtual）呼び出し規則でメソッドを呼び出します:
  • インスタンスコンストラクタ（<init>）
  • private インスタンスメソッド
  • 明示的なスーパークラスメソッド呼び出し（例: super.m()）
この命令は通常の virtual ディスパッチ探索をバイパスします。
オペランド:
  • methodref（2 バイト）: Methodref を指す定数プールインデックス
形式:
invokespecial <methodref>
Methodref format:

メソッド参照形式は，対象メソッドを次のように指定します。

<classname>-><methodname>(<argument_descriptor>)<return_descriptor>
  • <classname>: JVM 内部形式（スラッシュ区切り）の完全修飾クラス名。例: java/lang/Object
  • <methodname>: インスタンスメソッド名，または <init>（コンストラクタ）
  • <argument_descriptor>: JVM メソッド引数記述子
  • <return_descriptor>: JVM 戻り値型記述子
例:
  • invokespecial java/lang/Object-><init>()V
  • invokespecial my/pkg/Base->doWork()V（super 呼び出し）
  • invokespecial my/pkg/MyClass->privateHelper(I)V
スタック効果:

Before: ..., objectref, arg1, arg2, ..., argN
After: ..., returnValue?

受け手参照と全引数をポップします。戻り型が void でない場合は戻り値をプッシュします。 コンストラクタ呼び出しの戻り型は V です。

検証:
  • 検証では，未初期化オブジェクトに対するコンストラクタ規則および <init> の適法利用が強制されます。
備考:
  • <clinit> はバイトコード呼び出し命令で直接呼ばれず，クラス初期化として暗黙的に実行されます。
  • 意味的に非 virtual 選択が必要な場合は invokespecial を使用します。
実行時例外とエラー:
  • NullPointerException: 受け手オブジェクト参照が null の場合。
  • LinkageError: シンボリックメソッド参照を解決できない場合，または解決先メソッドにアクセスできない場合。
  • IncompatibleClassChangeError: 解決先メソッドが static の場合，コンストラクタ参照に対して不正なインスタンスメソッドが解決された場合，または呼び出し型に対して無効なメソッドハンドルの場合。
  • IllegalAccessError: 解決先インスタンスメソッドが現在クラスからアクセス不可の場合。
