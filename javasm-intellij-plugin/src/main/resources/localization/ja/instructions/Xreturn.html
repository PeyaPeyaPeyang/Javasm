メソッドからの復帰

{link:ireturn} - 0xac (172)
{link:lreturn} - 0xad (173)
{link:freturn} - 0xae (174)
{link:dreturn} - 0xaf (175)
{link:areturn} - 0xb0 (176)
{link:return} - 0xb1 (177)

目的

現在メソッドの実行を終了し，戻り型に応じて呼び出し元へ値を返します。各バリアントは次の戻り型に対応します。

  • {link:ireturn}: int を返す
  • {link:lreturn}: long を返す
  • {link:freturn}: float を返す
  • {link:dreturn}: double を返す
  • {link:areturn}: 参照（オブジェクトや配列）を返す
  • {link:return}: 戻り値なし（void メソッド）で復帰する
オペランド:

なし。返却する値は事前にオペランド・スタックへ積まれている必要があります。

形式:
{instruction}
例:
  • ireturn - オペランド・スタック先頭の int 値を返す
  • areturn - オブジェクトまたは配列参照を返す
  • return - void メソッドを終了する
スタック効果:
  • ireturn, freturn, areturn
    Before: ..., value
    After: （メソッド終了。値は呼び出し元へ渡される）
  • lreturn, dreturn
    Before: ..., value（カテゴリ 2）
    After: （メソッド終了。64 ビット値を呼び出し元へ渡す）
    備考: long と double はスタックスロットを 2 つ消費します。
  • return
    Before: ...
    After: （戻り値なしでメソッド終了）
備考:
  • 同期化メソッドから戻る場合，メソッドモニタは暗黙に解放されます。
  • 例外伝播中は return 命令は実行されず，例外処理側へ制御が移ります。
  • areturn はユーザー定義オブジェクトだけでなく配列（例: String[]，int[]）にも使用されます。
  • 通常復帰は現在フレームを終了し，呼び出し元継続点へ制御を戻します。
検証:
  • この命令位置で，戻り型に対応するカテゴリ値をオペランド・スタックが受け入れ可能であることが必要です。
  • return では，メソッド戻り型が void であることが必要です。
実行時例外とエラー:
  • IllegalMonitorStateException: 同期化メソッドで，現在スレッドが対象モニタを保持していない状態で return が実行された場合。
